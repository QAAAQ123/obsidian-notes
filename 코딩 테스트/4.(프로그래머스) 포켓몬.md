## 1. 문제 이해
- 절반만 가져갈 수 있음
- 같은 포켓몬이 여러 마리 일 수 있음
- 최대한 많은 종류의 포켓몬 선택
- 종류의 번호 return 

- nums: 포켓몬 종류,항상 짝수,최대 1만

## 2.복잡도 계산
- 10000 => 100,000,000
- N^2 이 1억

## 3. 자료구조 및 알고리즘
1. 배열x,스택x,트리x,그래프x
2. Map 가능
- Map
	- put한다. put할 때 중복 허용해서 중복이 많을 수록 value값 상승하게
	- x
- HashSet: 전체 다 넣고 set에 들어간 개수/2한 값
	- 중복이 알아서 걸러짐

## 4. 복잡도 검증
- 넣을때 N
- 길이 찾을 때 1
- -> O(N)

## 5. 로직 설계
1. 모든 데이터를 HashSet에 넣는다
2. 입력 배열 /2 >= set.size()이면 set.size리턴
3. else 입력 배열 / 2 리턴
