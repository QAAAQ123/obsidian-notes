1. List
	1. ArrayList: 크기가 변하는 배열
2. Set
	1. HashSet: 가장 빠름 순서 없음
	2. TreeSet: 데이터가 정렬됨($O(log N)$)
3. Map
	1. HashMap: 가장 빠름
	2. TreeMap: Key 기준 정렬
4. Deque
	1. ArrayDeque: Stack,Deque
5. Queue
	1. PriorityQueue: 최소,최대 힙
	2. ArrayDeque: Queue


| **분류**        | **인터페이스 (Interface)** | **구현체 (Implementation)** | **특징 및 용도**                                  | **시간 복잡도**   |
| ------------- | --------------------- | ------------------------ | -------------------------------------------- | ------------ |
| List          | **List**              | **ArrayList**            | **가장 기본.** 인덱스로 조회할 때 빠름. 배열과 유사.            | 조회 $O(1)$    |
|               |                       | **LinkedList**           | 삽입/삭제가 빈번할 때 유리하나, 코테에선 잘 안 씀.               | 삽입/삭제 $O(1)$ |
| Set           | **Set**               | **HashSet**              | **가장 빠름.** 중복 제거용. 순서 뒤죽박죽.                  | $O(1)$       |
|               |                       | **TreeSet**              | **오름차순 정렬**된 상태로 저장됨.                        | $O(\log N)$  |
|               |                       | **LinkedHashSet**        | 데이터가 **입력된 순서**대로 저장됨.                       | $O(1)$       |
| Map           | **Map**               | **HashMap**              | **가장 빠름.** 키(Key) 찾기용. 순서 뒤죽박죽.              | $O(1)$       |
|               |                       | **TreeMap**              | **키(Key) 기준 정렬**된 상태로 저장됨.                   | $O(\log N)$  |
|               |                       | **LinkedHashMap**        | 데이터가 **입력된 순서**대로 저장됨.                       | $O(1)$       |
| Stack/Queue   | **Deque**             | **ArrayDeque**           | 스택(LIFO)과 큐(FIFO) 기능 모두 수행. `Stack` 클래스 대체재. | $O(1)$       |
| PriorityQueue | **Queue**             | **PriorityQueue**        | **최솟값/최댓값**이 무조건 먼저 나옴(힙).                   | $O(\log N)$  |
