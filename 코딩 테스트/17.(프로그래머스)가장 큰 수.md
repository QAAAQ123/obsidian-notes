#sort
## 1. 문제 이해
0 또는 양의 정수가 주어졌을 때, 정수를 이어 붙여 만들 수 있는 가장 큰 수를 알아내 주세요.

예를 들어, 주어진 정수가 [6, 10, 2]라면 [6102, 6210, 1062, 1026, 2610, 2106]를 만들 수 있고, 이중 가장 큰 수는 6210입니다.

0 또는 양의 정수가 담긴 배열 numbers가 매개변수로 주어질 때, 순서를 재배치하여 만들 수 있는 가장 큰 수를 문자열로 바꾸어 return 하도록 solution 함수를 작성해주세요.

- 조합할 수 있는 모든 경우의 수 중에서 가장 큰수 리턴
## 2.입력 크기 및 시간/메모리 제한 확인
- numbers의 길이는 1 이상 100,000 이하입니다.
- numbers의 원소는 0 이상 1,000 이하입니다.
- 정답이 너무 클 수 있으니 문자열로 바꾸어 return 합니다.
- N=100,000(십만)=$10^5$
- $N^2=10^{10} -> N^2$은 불가능
- $O(N\log{N})$만 가능
## 3. 자료구조 및 알고리즘
- 커스텀 정렬 알고리즘
## 4. 로직 설계
- 우선순위
	- 최대 자리 숫자가 클수록 우선순위 높음
	- 최대 자리 숫자가 크면 자릿수가 클 수록 높음
	- `3, 30, 34, 5, 9`: 9,5,34,30,3순임
	- "9534303"|"9534330" -> 아님
- `3, 30, 34, 5, 9`
1. `3, 30, 34, 5, 9`(3,30비교)
2. `3, 34, 30, 5, 9`(34,30비교)
3. `3, 34, 5, 30, 9`(5,30비교)
4. `3, 34, 5, 9, 30`(30,9비교)
- 전체를 문자열 배열로 만들고 sort로 정렬 기준 제시해줌=> 가능,시간복잡도도 $O(N \log{N})$임 
**추가: 문자열 전체가 0일때는 00000이 아닌 0을 리턴해줘야함**
## 5. 복잡도 검증
- int를 string으로 만들때: O(N)
- A와 B를 합치고 비교할 때: O(1) -> N번: O(N)
- sort() 복잡도: $O(\log_2{N})$ 
- 결론: $O(N\log{N})$

## 6. 라이브러리 선택
- Arrays.sort 사용
- String
## 7. 검증
```java
import java.util.*;
class Solution {
    public String solution(int[] numbers) {
        //담기
        String[] bucket = new String[numbers.length];
        for(int i = 0; i< numbers.length;i++){
            bucket[i] = String.valueOf(numbers[i]);
        }
        
        //정렬
        Arrays.sort(bucket,(a,b) -> (b + a).compareTo(a+b));
        
        if(bucket[0].equals("0")) return "0";
        
        
        StringBuilder sb = new StringBuilder();
        for(String s:bucket) sb.append(s);
        
        return sb.toString();
    }
}
```
## 8. 몰랐던 메소드 정리
- String.valueOf(x): x를 String으로 변환
- (x).compareTo(y): x와 y를 비교
- Arrays.sort을 이용해 정렬 조건을 커스텀할 수 있음
- Arrays.sort(bucket,(a,b) -> (b + a).compareTo(a+b))
	- Arrays.sort(bucket): 기본 정렬
	- Arrays.sort(bucket,정렬 조건(Comparator클래스)): 커스텀 정렬
	- (a,b) -> (b + a).compareTo(a+b)에 원래 클래스를 써줘야 하는데 람다식 형태로 간편하게 쓴것
- StringBuilder의 타입이 String이 아니다. 따라서 StringBuilder로 조립을 완료한 후에 toString으로 변환해줘야 한다.
