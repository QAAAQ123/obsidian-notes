## 1. 문제
Leo는 카펫을 사러 갔다가 아래 그림과 같이 중앙에는 노란색으로 칠해져 있고 테두리 1줄은 갈색으로 칠해져 있는 격자 모양 카펫을 봤습니다.

![carpet.png](https://grepp-programmers.s3.ap-northeast-2.amazonaws.com/files/production/b1ebb809-f333-4df2-bc81-02682900dc2d/carpet.png)

Leo는 집으로 돌아와서 아까 본 카펫의 노란색과 갈색으로 색칠된 격자의 개수는 기억했지만, 전체 카펫의 크기는 기억하지 못했습니다.

Leo가 본 카펫에서 갈색 격자의 수 brown, 노란색 격자의 수 yellow가 매개변수로 주어질 때 카펫의 가로, 세로 크기를 순서대로 배열에 담아 return 하도록 solution 함수를 작성해주세요.

## 2. 문제 이해
- 테두리 하나만 갈색이다.
- 노란색은 몇줄이든 상관없다
- 정답은 하나이다

- ~~노란색이 정해지면 갈색은 알아서 결정됨 or 갈색이 정해지면 노란색은 알아서 결정됨~~
- ~~갈색의 격자의 수가 적으니 갈색의 가로,세로 길이를 찾고 그 안에 노란색이 들어가는지 확인~~
- $노란색+갈색=x\times y$인 x,y짝(약수)을 찾아서 갈색이 특정 x,y에 테두리 개수에 맞는지 확인하면 됨(단 x>=y)일때

## 3.입력 크기 및 시간/메모리 제한 확인
- 갈색 격자의 수 brown은 8 이상 5,000 이하인 자연수입니다.
- 노란색 격자의 수 yellow는 1 이상 2,000,000 이하인 자연수입니다.
- 카펫의 가로 길이는 세로 길이와 같거나, 세로 길이보다 깁니다.

- `가로==세로 || 가로 > 세로`

## 4. 최대 시간복잡도 확인
- Brown = 5000
- yellow = 2,000,000=$2\times10^6$
 - $Brown^2 = 25,000,000이고 \log_{2}5000=\log_{2}(5\times 10^3)이다. 따라서 Brown^2까지 가능하다$
 - yellow는 $yellow와log(yellow)$까지 가능

## 5. 자료구조 및 알고리즘
- brute-force
- hashset

## 6. 로직 설계
- hashset선언
- total = brown + yellow
- while(x>=y)
	- if($total \% x == 0$)
		- y = total / x
		- if(brown = 2(x+y-2))
			- return x,y
	- x--

## 7. 시간복잡도 검증
- while문: 2,005,000의 절반의 시간 복잡도
- O(yellow + brown)=> 충분

## 8. 코드 검증
```java
class Solution {
    public int[] solution(int brown, int yellow) {
        int total = brown + yellow;
        int x = total-1;
        int y = 1;
        
        while(x >= y){
            if(total % x == 0){
                y = total / x;
                if(brown == 2 * (x+y-2)) return new int[]{x,y}; 
            } 
            x--;
        }
        
        return null;
    }
}

        
```
