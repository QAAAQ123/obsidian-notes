61. CNI: **Cilium**-네트워크 정책
	1. 고급 네트워킹, 보안 및 관찰 기능을 제공하는 **eBPF 기반** 클라우드 네이티브 솔루션 쿠버네티스 워크로드
	2. **~={red}layer7에서 동작=~**(HTTP,DNS 등)
	3. 로드 밸런싱을 위해서 최적화된 eBPF 구현
62. CNI: **Calico**-네트워크 정책
	1. eBPF기반
	2. BGP 라우팅 프로토콜을 사용하는 **~={red}레이어 3에서 동작=~**하는 네트워킹 솔루션
	3. 기업에서 주로 사용
	4. 클러스터 전체에서 전송 중인 데이터를 안전하게 보호하기 위해 WireGuard 암호화를 제공합니다
63. eBPF란?-네트워크 정책
	1. 확장 버클리 패킷 필터
	2. Linux 커널 공간 내에서 바이트코드 프로그램을 안전하게 실행 하는 커널 내 가상 머신-커널에서 돌아가게 프로그래밍한 코드
	3. 런타임에 즉각적으로 컴파일해서 바이트코드를 샌드박싱 형태로 실행
	4. **커널에서 원하는 코드를 수행하게 해주는 메커니즘**
	5. 커널에서 프로그래밍 하기 때문에 빠르고 위험하다
	6. 이벤트 기반 프로그래밍-이벤트를 발생하면 eBPF가 동작
64. eBPF를 통한 정책 실행-네트워크 정책
	1. eBPF 맵에 어떤걸 허용할지 저장해두고 eBPF 프로그램이 맵의 정보를 확인해서 ACTION을 실행한다
	2. deny이면 TC hook에서 패킷을 버린다
	3. 이걸 eBPF가 커널에서 해준다
	4. 유저 모드에서 이걸 하면 성능이 떨어진다
65. CNI 솔루션 간 네트워크 정책 구현 비교-네트워크 정책
