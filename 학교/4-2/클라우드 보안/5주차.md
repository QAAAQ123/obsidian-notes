74. ZTA의 핵심 컴포넌트 4개 정의
	1. PDP: 정책 결정-**~={red}Decision Point=~**
	2. PEP: 리소스로 가는 최종 관문(PDP 결정 실행)-**~={red}Enfocement Point=~**
	3. PA: 주체와 자원 사이의 통신 경로 설정/해체-동적 권한 부여를 위해 PEP와 연결됨-Administrator
	4. PE: 신뢰 알고리즘 기반으로 승인/거부/취소 결정을 내리는 컴퓨팅 엔진-Policy Engine
	5. 오른쪽 그림이 동작 구조
		1. 4개의 입력을 Control Plane으로 들어가서 
		2. PE와 PA가 판단을 내림
		3. Data Plane에 있는 PEP를 통해서 실제 접근 제어로 구현

		4. 이식성: 쉽게 다른 환경으로 이동가능

	1. 4C모델로 접근: 클라우드 보안을 4개의 계층으로 나눠서 생각-유기적으로 환경을 아우르면서 보안 
		1. Cloud 계층: 인프라 보안과 관련(IaaS,PaaS)
		2. Cluster 계층: 컨테이너 오케스트레이션 플랫폼(K8S,OpenShift)-클러스터 자체 보안
		3. Container 계층: 런타임,이미지,레지스트리
		4. Code 계층: 실시간 앱,의존성, 비즈니스 로직-시크릿 관리(마이크소 서비스간 데이터 노출 하지 않는 것이 핵심)
