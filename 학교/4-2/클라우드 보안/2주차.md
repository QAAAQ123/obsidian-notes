1. 구름과 클라우드 컴퓨팅 비교
2. 클라우드 컴퓨팅이란?
	1. NIST: 키워드 4개
		1. on-demand: 사용자가 원할때 바로 리소스를 제공받는것
		2. configurable: 원하는 사양에 맞게 자원 할당
		3. rapidly provisioned: 빠르게 클라우드 서비스를 할당 받음
		4. 최소 관리 노력: 자동화
	- 인터넷 연결된 상황에서 내가 원하는 컴퓨팅 자원을 원하는 때에 원하는 양만큼 사용하는 개념
3. 
	1. 엔드유저와 서비스 제공 사용자 간에 **인터넷**이 연결되어있어야 한다.
	2. 언제 어디서든 접속 가능
	3. 거대한 데이터 센터로 부터 인터넷을 연결해서 집까지 remote로 사용하는 것
4. 클라우드 컴퓨팅 benefit
	1. 유연성
	2. 비용 절약(사용한 양 만큼 비용 청구)-물리적으로 구축할 때보다 초기비용 대폭 감소
	3. 보안
	4. 확장성
	5. 쉬운 백업 및 복구
	6. Reliability: 신뢰성
	7. 등등
5. 클라우드 컴퓨팅 단점
	1. Vender Lock-in: 다른 서비스로 교체할 떄 교체 비용이 큼
	2. Cost management: 초기 구축에 절감되지만, 사용량에 따라 요금 부과하기 때문에사용량 가늠하기 어려움/비용 관리
	3. 보안,개인정보: 클라우드도 물리적 저장소에 저장하기 때문에 보안 위험이 존재한다.
	4. shared infraxx... risk: 공유하는 형태이기 때문에 문제 발생 소지가 있음
	5. service downtime: 서버 셧다운 되면 서비스 다운 시간이 있을 가능성 있음
	6. ㅇㅇ
