## AWS 대표 스토리지 서비스 요약 
- **Amazon S3 (Simple Storage Service)**:
    
    - 파일 기반의 저장/검색/삭제 서비스입니다1.
        
    - '오브젝트 스토리지'라고도 불립니다2.
        
    - 무제한 확장성을 제공합니다3.
        
    - 웹 기반 인터페이스를 지원하며, 공개적 또는 비공개적 접근이 가능합니다4.
        
    - EBS나 EFS에 비해 성능(속도)은 느린 편입니다5.
        
    - 주요 사용 사례: 엔터테인먼트/미디어, 빅데이터 분석, 아카이브 및 백업, 콘텐츠 관리 및 웹 서빙6.
        
- **Amazon EFS (Elastic File System)**:
    
    - 폴더와 파일 기반의 계층적 구조를 가집니다7.
        
    - 파일 스토리지 유형입니다8.
        
    - 공유 파일 시스템 워크로드를 필요로 하는 환경에 호환됩니다9.
        
    - 무제한 확장성을 제공합니다10.
        
    - 파일 시스템 인터페이스를 지원하며, 여러 EC2 인스턴스 및 온프레미스 인스턴스에서 동시에 접근 가능합니다11.
        
    - S3보다는 빠르고 EBS보다는 느린 성능을 가집니다12.
        
    - 주요 사용 사례: 엔터테인먼트/미디어, 빅데이터 분석, 아카이브 및 백업, 콘텐츠 관리, 홈 디렉토리 및 웹 서빙13.
        
- **Amazon EBS (Elastic Block Store)**:
    
    - 데이터베이스 가동 시간 속도 등에 민감한 블록 스토리지 서비스입니다1414.
        
    - 읽기/쓰기 속도에 최적화된 스토리지 서비스입니다15.
        
    - 블록 스토리지 유형입니다16.
        
    - 확장성은 수동으로 볼륨 크기를 관리해야 합니다17.
        
    - 연결된 EC2 인스턴스를 통해서만 접근 가능합니다18.
        
    - S3나 EFS에 비해 성능(속도)이 가장 빠릅니다19.
        
    - 주요 사용 사례: 볼륨 부팅, NoSQL 및 트랜잭션 데이터베이스, ETL 기반 데이터 웨어하우징 요구사항20.
        
- **Amazon S3 Glacier**:
    
    - 접근 빈도가 낮은 데이터의 장기 아카이브 스토리지입니다21.
        
    - 디지털 정보 저장 및 데이터 테이프 교체용으로 사용됩니다22.

## 파일 스토리지 vs 블록 스토리지

**공통점:**

- PDF 내에서 파일 스토리지(EFS)와 블록 스토리지(EBS)의 직접적인 공통점에 대한 명시적인 언급은 찾기 어렵습니다. 둘 다 클라우드에서 데이터를 저장하는 기술이라는 점 외에는 구체적인 공통 설명이 부족합니다.
    

**차이점:**

|**구분**|**파일 스토리지 (EFS 예시)**|**블록 스토리지 (EBS 예시)**|
|---|---|---|
|**저장 방식**|폴더와 파일 기반의 계층적 구조 23|데이터를 고정된 크기의 블록 단위로 저장 (PDF 내 직접 언급 부족)|
|**접근 방식**|파일 시스템 인터페이스 24, 여러 서버에서 동시 접근 가능 25|연결된 단일 EC2 인스턴스를 통해 접근 26|
|**주요 용도**|공유 파일 시스템 27, 웹 서빙, 홈 디렉토리 28|OS 부팅, 데이터베이스, 고성능 요구 작업 292929|
|**성능**|S3보다 빠르고 EBS보다 느림 30|S3, EFS보다 빠름 31|
|**확장성**|무제한 확장성 제공 32|수동으로 볼륨 크기 관리 필요 33|
