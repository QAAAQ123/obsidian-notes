## ☁️ 클라우드 물리 인프라 계층 구조

클라우드 서비스는 전 세계에 분산된 물리적 인프라를 기반으로 제공됩니다. 이 인프라는 일반적으로 다음과 같은 계층 구조를 가집니다:

1. **데이터 센터 (Data Center)**
    
    - **정의:** 서버, 스토리지, 네트워킹 장비 등 IT 인프라를 수용하는 물리적인 건물 또는 시설입니다. 전용 전력, 냉각, 네트워크 인프라를 갖추고 있습니다.
        
    - **역할:** 클라우드 서비스를 실행하는 데 필요한 컴퓨팅 자원의 물리적 기반을 제공합니다.
        
    - **계층:** 가장 기본적인 물리적 단위입니다.
        
2. **가용 영역 (Availability Zone, AZ)**
    
    - **정의:** 하나의 리전 내에서 물리적으로 분리된 하나 이상의 데이터 센터 그룹입니다. 각 AZ는 독립적인 전력, 냉각, 네트워킹을 갖추고 있어 서로 격리되어 운영됩니다. AZ 간에는 낮은 지연 시간(Low Latency)의 네트워크로 연결됩니다.
        
    - **역할:** 특정 데이터 센터나 AZ 전체에 장애가 발생하더라도 다른 AZ에서 서비스를 계속 제공하여 **고가용성**과 **내결함성**을 보장합니다.
        
    - **계층:** 데이터 센터 < **AZ**
        
3. **리전 (Region)**
    
    - **정의:** 하나 이상의 가용 영역(AZ)을 포함하는 넓은 지리적 영역(예: 서울, 도쿄, 버지니아 북부)입니다. AZ들은 리전 내에서 서로 낮은 지연 시간 네트워크로 연결됩니다.
        
    - **역할:** 지리적 위치에 따른 서비스 지연 시간 감소, 데이터 주권 및 규정 준수 요구사항 충족, 재해 복구 전략의 기반을 제공합니다.
        
    - **계층:** 데이터 센터 < AZ < **리전**
        

**요약:** 여러 **데이터 센터**가 모여 하나의 **가용 영역(AZ)**을 구성하고, 여러 **AZ**가 모여 하나의 **리전(Region)**을 형성하는 계층 구조를 가집니다. 사용자는 특정 리전과 그 안의 AZ를 선택하여 클라우드 리소스를 배포합니다.

## VPC (Virtual Private Cloud) 개념

VPC는 각 고객에게 독립적인 클라우드 환경을 제공하기 위해 생성하는 가상 네트워크입니다1. 이는 **~={red}논리적으로 격리된 공간=~**을 만들어 사용자가 자체 네트워크 환경을 구성하고 제어할 수 있게 해줍니다. 주요 특징은 다음과 같습니다:

- **독립적인 환경**: 각 고객별로 분리된 네트워크 환경을 제공합니다2.
    
- **네트워크 구성**: 사용자는 사설 IP 주소 범위(서브넷)를 정의하고 인터넷 게이트웨이를 설정할 수 있습니다333.
    
- **보안 제어**: 방화벽 및 네트워크 접근 제어 기능(예: '보안 그룹'을 통한 트래픽 필터링)을 제공합니다444.
    
- **DNS 통합**: DNS 서비스와 통합됩니다5.

## VPC 사용 방법 (구성 요소 기반)

PDF에서는 VPC를 '어떻게 사용하는지'에 대한 단계별 절차보다는, VPC가 어떤 구성 요소들로 이루어져 어떻게 구조화되는지를 설명합니다. 이를 통해 사용 방법을 유추할 수 있습니다.

1. **VPC 생성**: 클라우드 내에 논리적으로 격리된 가상 네트워크 공간을 만듭니다6.
    
2. **서브넷(Subnet) 구성**: VPC의 IP 주소 범위를 더 작은 단위로 나누어 서브넷을 생성합니다777. 필요에 따라 외부 인터넷과 통신할 수 있는 '퍼블릭 서브넷'과 외부에서 직접 접근할 수 없는 '프라이빗 서브넷' 등으로 구성할 수 있습니다8888.
    
3. **라우터(Router) 및 라우팅 테이블 설정**: 서브넷 간의 트래픽 경로와 외부 통신 경로를 라우팅 테이블을 통해 정의합니다999999999.
    
4. **게이트웨이(Gateway) 연결**: 필요에 따라 인터넷 게이트웨이 등을 연결하여 VPC 내부와 외부(인터넷 등) 간의 통신을 가능하게 합니다101010101010101010.
    
5. **리소스 배치**: 생성된 VPC 및 서브넷 내부에 가상 서버(예: EC2 인스턴스)와 같은 클라우드 리소스를 배치합니다11111111111111111111111111111111.
    
6. **보안 설정**: 보안 그룹이나 네트워크 ACL 등을 이용하여 리소스에 대한 인바운드/아웃바운드 트래픽을 제어합니다12.


## VLAN vs. SDN 비교

VLAN과 SDN은 모두 네트워크를 **논리적으로 관리**하고 **유연성**을 높이는 기술이라는 공통점이 있습니다.

**주요 차이점:**

- **VLAN:** 주로 **Layer 2**에서 스위치 기반으로 **네트워크(브로드캐스트 도메인)를 분할**하는 기술입니다.
    
- **SDN:** 네트워크의 **제어 기능과 데이터 전달 기능을 분리**하여, **소프트웨어(중앙 컨트롤러)**를 통해 전체 네트워크를 **프로그래밍하고 중앙에서 관리**하는 아키텍처입니다.
