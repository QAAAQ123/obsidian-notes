무조건 나옴
클라우드 보안 핵심 원칙 5가지
	1. 가용성: 사용자 언제 어디에서든 클라우드에 접근 가능해야 한다
	2. 기밀성: 사용자의 데이터를 비밀로 유지해야 한다,
	3. 데이터 무결성: 정보 무결성 보존,승인되지 않은 사용자에 의해 손실,수정되지 않아야 함
	4. 통제(컨트롤): 사용자의 데이터 사용과 노출에 대한 우려,
	5. 감시: 클라우드 환경을 모니터링,로그 분석

클라우드 보안 VIA 프레임워크
1. 가시성: 보안 위협과 보안 위험이 투명하게 보여야 함-시각화 
2. 인텔리전스: 기존의 오류와 지연을 극복하고 빠른 예측과 분석
3. 오토네이션(자동성): 사고 발생시 보호 조치가 지연없이 즉시 적용

하이퍼바이져 지향점
1. 충성도(Fidelity): 게스트 OS입장에서 물리 머신의 OS와 똑같이 가상 머신이 동작해야 한다
2. 격리와 안전: 일반적으로 여러개의 VM을 사용하기 때문에 각각의 VM을 완전히 격리시켜줘야 한다.(Host OS의 접근도 허용하지 않아야 한다)
3. 성능: 하이퍼바이져를 거쳐야 하기 때문에 물리에서 돌아가는 OS보다는 성능 저하가 있을 수 밖에 없다.이 성능 저하를 최소화 하는 것
