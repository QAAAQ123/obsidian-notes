## 접근 컨트롤
- 정의: 어떤 주체가 어떤 객체에 대해서 접근 행동을 할지 제어하는 기능
- 과정: 유저 -> 인증 -> 접근 제어 기능 -> 리소스 접근
	- 인증: 유저A가 유저 A라는 것을 확인
	- 보안 관리자<-->인가 DB<--> 접근 제어 기능
- 인증: 유저가 유저가 맞는지 확인/인가: 유저에게 리소스의 접근 권한을 부여

## 인증과 접근 제어
- 식별: 유저ID,GID등
- 인증: pw,지문,홍채,얼굴,목소리 등
- 접근 제어: 자원에 대한 접근을 선택적으로 하게 해줌
- 인가: 주체를 정책에 따라 리소스에 접근 가능한지 결정

## 접근 제어 개요
1. access: 주체와 객체 사이의 정보 흐름
2. 접근 제어 기본 요소
	1. 주체
		1. 주체는 프로그램,시스템도 가능하다
	2. 객체
		1. 통신 포트,블록,페이지 등.. 리소스
	3. 접근 권한
3. 접근 제어 필요한 이유
	1. 기밀성: 다른 사용자가 읽는 것 거부 가능해야 한다
	2. 무결성: 파일이 훼손되지 않았다는 것을 증명해야 한다

## 접근 제어 요구 사항
1. 최소 권한
	1. 필요한 최소한의 권한으로 작업
	2. 피해 최소화
2. 권한 분리
	1. 여러 사용자에게 권한을 분할하여 작업 수행
	2. 피해가 다른 서비스에 영향을 미치지 않아야 한다
3. Dual control
	1. 두 사람이상이 허용해야만 실행하거나 접근 허락
	2. 위험하거나 중요한 활동에 활용

## 접근 제어 모델
1. DAC: 소유자가 리소스 권한 지정(리눅스 rwx모델)
2. MAC: 중앙에서 라벨 기반으로 모든 파일의 접근 권한 통제
3. RBAC: 역할기반 접근 제어
