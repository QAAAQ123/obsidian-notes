## 네트워크 개요
1. 해저 케이블이 국가간 인터넷을 연결한다
2. 원래 IP: 클래스 기반-**~={red}32bit=~**
	1. a,b,c,d,e
3. CIDR: 기존 IP가 부족해서 나왔다
	1. 네트워크ID를 나타내는 비트
	   ex) /24: 32비트중 24비트가 네트워크 식별하고 나머지 8비트가 개별기기 식별
	2. **~={red}질문:192.168.192.0/19의 주소 범위-시험=~**
		1. 19비트가 네트워크 식별한다. 
		2. 10진수 3자리는 2진수로 8자리이다
		3. 앞에서부터 19비트까지 고정 xxxxxxxx.xxxxxxxx.xxx00000.00000000
		4. x는 고정 0은 변동
		5. 192=11000000
		6. 따라서 11000000.00000000부터 11011111.00000000까지 입니다
		7. 10진수로는 192.0부터 223.255까지
4. 특수 IP
	1. 사설 IP 주소: 내부 네트워크 안에서만 사용하기로 예약된 대역
	2. 루프백 주소: 자기 자신
		1. 127.0.0.0/8
5. NAT: 사설 IP를 사용하는 여러 대의 내부 컴퓨터가 하나의 공인 IP(게이트웨이)를 공유하여 인터넷을 사용할 수 있게 해주는 기술
	1. 요청: 내부 컴퓨터가 요청을 하면 패킷을 게이트웨이로 보냄
	2. 주소 변환: 게이트웨이는 공인IP로 바꿔서 내보냄
	3. 응답: 응답이 돌아오면 공인IP를 요청했던 사설IP로 바꿔서 컴퓨터로 전달
6. ip -br address: ip보는 명령어
7. DHCP: IP 자동 할당
8. DNS:외우기 힘들어서/IP바꿔도 그대로 접속 가능

## 네트워크 보안
1. 공격
	1. 과저: 자가복제 멀웨어
	2. 과정: 탐색 -> 감염
	3. 현재: drive-by download,bot,DOS 등...
2. 바이러스와 worm의 차이
	1. worm: 스스로 전파,본인을 복제,악의적이거나 악의적이지 않음
	2. 바이러스: 공격 대상이 분명하고 항상 악의적
3. worm 사례
	1. 모리스 worm: 6000대-BOF 이용
	2. 코드 레드 worm: 2000년 500,000대
		1. MS IIS 웹 서버 공격
4. worm 상태 전이: 기생충 전파 방식
	1. 감염 가능,감염됨,제거됨
	2. 감염된 호스트가 다른 호스트 감염 시킴
	3. 제거된 호스트: 패치나 감염 경로에서 빠진 컴퓨터

## 네트워크 공격
- 무결성,기밀성,가용성 공격
- TCP와 UDP
	- TCP: SYN sacn->SYN보내서 되돌아 오면 해당 포트가 열려있다고 판단
	- UDP: 랜덤 포트 요청-열려 있으면 반응이 없고, 닫혀있으면 unreachable 응답을 받는다
	- ICMP: 호스트에 접근 할 수 있는지 확인하는 프로토콜
- NMAP: 스캐닝 툴
	- victim 입장에서 스캔닝 방어
		- 포트 스캔: 정상 사용자는 포트를 한개만 요청/여러 포트에 의심스러운 패킷을 보낸 IP를 차단함
		- OS 추측하는것 해결법: TTL 수정해서 응답 패킷 보냄
			- TTL: IP 헤더 값, 라우터 무한 회전 방지
				- 라우터 지날때마다 1감소
## 고도화된 기술
1. drive-by download: 악성 소프트웨어가 있는 서버로 연결하는 방식,추적 최소화해야함
	1. 사용자가 해킹당한 사이트에 접속
	2. 리다이렉트 명령
	3. 경유지(Hop point)로 접속
	4. 경유지에서 다시 리다이렉팅
	5. 유포지에 접속하여 악성코드 다운로드 
	6. 실행
2. BOT
	1. 특정 행위를 자동으로 수행하는 애플리케이션
		1. 상업적 목적: 웹 크롤링
		2. 악의적 목적: 봇 넷
3. Botnet
	1. bot: 봇에 의해 손상된 컴퓨터
	2. botnet: 공격자가 제어하는 봇 네트워크
	3. 공격 과정
		1. 기기에 심어놓거나 설치
		2. 공격 요구자에게서 금전을 받음
		3. 고객이 원하는 행동을 해줌
4. Dos(Denial of Service)
	1. TCP SYN flooding 공격: SYN 패킷을 SYN 플래그 ON해서 보냄
	2. TCP의 3-hand shacking: SYN -> SYN ACK -> ACK
	3. ACK에서 클라인트 정보를 알고 있어야 함->서버에서 SYN 패킷 정보 저장하는 버퍼 만들어야 함
	4. 버퍼가 다 차게 무수한 공격을 보내서 정상 유저의 접속을 막아버린다
	5. 분산된 시스템에서 Dos를 하면 DDos
5. 트로이 목마: 유용한 기능을 가진 것처럼 보이지만, 보안 메커니즘을 회피하는 숨겨진 잠재적 악의적 기능을 가진 컴퓨터 프로그램
6. 스팸 이메일: 트로이 목마가 들어있을 수 도 있음,각종 멀웨어
7. 피싱: 미끼를 사용하여 피해자를 낚음
