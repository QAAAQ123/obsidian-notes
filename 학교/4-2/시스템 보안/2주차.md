		2. 무결성: 위변조가 되지 않았음을 보장함 
			1. ex) Hash Function(단방향 Function):이 경우에는 Hash값 대조를 통해 무결성 보장
			2. 웹 페이지의 Hash와 다운로드 중 Hash와 비교를 통해 무결성 보장
		3. 가용성: 사용가능성
			1. 사용하지 못하게 공격(디도스,도스 공격)

35. OS에서의 보안
	1. 메모리에서 Stack은 자료구조 Stack과 동작은 같지만 세부 내용은 다르다
	2. 메모리에서 Stack으 동작: ~={red}함수가 호출 될 때=~ 함수 정보(파라미터,리턴 등)을 할당/함수가 끝나면 공간 다시 회수
	3. Stack Overflow Violation: 허용된 Stack 공간을 벗어나는 것
	4. 권한 상승으로 공격: 일반 유저 권한에서 super유저나 root유저로 권한을 상승시켜 공격 가능

	1. **악의적인 의도**를 가지고 수행하거나 **사고에 의해서** 일어나는 것
	2. Threat(위협): 잠재적인 보안 위반

	2. 불법 행위자: 정당한 사용자이지만, 권한 악용
	3. 은밀한 사용자: 염탐해서 정보를 캐냄

	1. Default Password 사용
	2. 무작위로 계속 뚫음
	3. 가장 많이 사용하는 패스워드 목록을 가지고 공격 시도
	4. 개인 정보 기반으로 조합해서 시도
	5. 트로이 목마: 평범한 프로그램인척 해서 설치하면 공격
39. 공격 방법
	1. 인터럽트:A->B로 보낼때 아예 막아서 훔침
	2. 훔치기:A->B로 보낼때 해커가 훔침
	3. 변조:A->해커/해커->B 방법을 통해 A는 B로 보내는것 처럼,B는 A에서 받는 것 처럼 보이게 함
	4. 위조: A가 없고 해커가 A인척하여 B에 접근
