## pytest
- assert()로 예상한 결과와 일치하는지 확인
- 테스트 파일과 함수에 `test_`를 붙혀야 테스트 코드인지 인식한다
- ex)test_router.py와 test_get_all_event()로 해야 test 코드로 인식
- pytest가 실행될때 루트 디렉토리에 있는 pytest.ini 설정 파일의 내용을 불러온다
### 픽스쳐
- 픽스쳐: 재사용할 수 있는 함수
- 테스트 함수에 필요한 데이터를 정의하기 위해 사용
- API 라우트 테스트스 애플리케이션 인스턴스를 반환하는 경우 등에 사용
- pytest.fixture 데코레이터로 사용
- 인수를 선택적으로 받을 수 있음
	- scope인수를 받아 유효범위 지정
		- session: 테스트 세션 동안 유지
		- module: 특정 함수에서만 유지
```python
import pytest

@pytest.fixture
def event() -> EventUpdate:
	return EventUpdate(
	title = "title",
	image = "image"
	)

def test_event_title():
	assert event.title = "title"
```

## 비동기 테스트
- 비동기를 테스트 하려면 httpx와 pytest-asyncio가 필요하다
- 비동기 설정을 pytest.ini에 `[pytest]\nasyncio_mode=auto`로 모든 테스트를 비동기로 하라고 알려준다
