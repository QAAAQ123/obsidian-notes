## 테이블
- 테이블 모델 클래스 정의해야 DB를 사용 할 수 있음
- 기본키는 Field()함수로 정의
### 데이터 저장 과정
1. 데이터를 하나로 만든 인스턴스 생성
2. 인스턴스를 Session 클래스에 넣음
3. Session을 넣은 객체를 트랜잭션 커밋하여 저장

## 세션
- 세션 객체는 코드와 DB 사이에서 이루어치는 처리 관리
- Session 클래스는 SQL 엔진의 인스턴스를 인수로 사용한다
- Session 메서드
	- add(): 데이터 객체를 메모리에 추가,커밋하기 전까지 대기
	- commit(): 현재 세션에 있는 트랜잭션 모두 정리
	- get(): 단일 Low 추출,모델과 문서ID를 인수로 사용

## SQL Model 사용
1. 기존 BaseModel을 SQLModel로 변경
2. DB connection
3. session 생성
	1.  yield: return문과 유사하지만 변수의 마지막 값과 상태를 저장
4. 메인에 DB연결
