## 1. 모듈
- 모듈: `*.py`를 다른 파일에서 import하면 모든 .py파일이 모듈이 된다.
- `import`문만 작성해서 코드를 쓰면 모든 경로를 코드에 작성해야 한다
- `from 패키지 import 모듈` 형태로 쓰면 모듈만 코드에 작성해 주면 된다.
```python
import mypack.calc.add
mypack.calc.add.outadd(1,2)

form mypack.calc import add
add.outadd(1,2)
```


### 모듈 테스트 코드
```python
INCH = 2.54

def calcsum(n):
	sum = 0
	for num in range(n+1):
		sum += sum
	return sum
	
if __name__ == "__main__":
	print("인치 =",INCH)
	print("합계 =",calcsum(10))
```
- 여기서 **~={red}`__name__`은 현재 실행 중인 모듈의 이름이다=~.** 
- 따라서 `__name__ == "__main__`은 현재 모듈이 메인인지 확인하는 것이다
- 위 코드는 ~={red}메인 모듈일 때 즉 **다른 모듈에서 import 되지 않고 직접 사용될 때만 True**인 테스트 코드이다.=~


## 2. 패키지

### `*`로 모든 모듈 import
- `form 패키지 import *`하면 패키지 내의 모든 모듈을 사용할 수 있다. 하지만 이렇게 하면 예상치 못한 부작용이 발생할 가능성이 많다. 메모리 과다 사용,메서드 이름 충돌 등등
- 그래서 **~={red}패키지 디렉토리에 `__init__.py`파일을 작성해서 `*`로 import하면 어떤 모듈을 불러올 것인지 지정해 둔다.=~**
```python
__all__ = ["add","multi"]
print("add,multi module imported")
```
- 파이썬은 한번 임포트한 모듈을 컴파일 한 상태로 캐시에 저장해둬서 필요할 때 별도의 로딩 과정 없이 바로 사용할 수 있다. 

### 컴파일된 파이썬의 확장자
- .py가 컴파일된 확장자는 .pyc이다. 
- 이 pyc는 파이썬 버전별로 생성되어서 한 모듈을 각각 다른 파이썬 버전에서도 사용할 수 있다.

## 3. 서드 파티 모듈
- 내부 모듈과 표준 모듈은 매우 많다. 
- `dir(패키지)`를 하면 내부의 모듈을 모두 볼 수 있다. 
- 서드 파티 모듈도 많다. Flask,pyGame 등 많다. 
- 서드 파티 모듈은 pip(Python Package Index)로 설치한다.
