### 커맨드 분류
1. 이미지 관리
2. 컨터이너 관리
3. 실행중인 컨테이너와 상호작용
4. 컨테이너 디버깅 커맨드

### 일반 커맨드
1. docker --version
2. docker info
3. docker help

### 이미지 관리
1. docker images: 로컬 머신의 이미지 목록
2. docker pull 'image-name': docker hub나 레지스트리에서 이미지를 받아 로컬의 캐시에 저장
3. docker build -t 'image-name' .: 현재 디렉토리를 중심으로 이미지를 만듦,`-t [image-name]`을 사용해서 image-name으로 태깅을 함
4. docker tag 'source-image' 'target-image:tag': `source-image`에 태깅함
5. docker push 'image-name': docker hub나 레지스트리에 push
6. dockr rmi 'image-id': 이미지 id기반으로 여러개의 삭제

### 컨테이너 관리
1. docker ps: 현재 실행중인 컨테이너 목록
2. docker ps -a: 모든 컨테이너 목록(멈춘 컨테이너까지)
3. docker run 'options' 'image-name'
	1. -d: 백그라운드에서 실행
	2. -p 'host-port':'container-port': 호스트 포트와 컨테이너 포트 연결
	3. --name 'container-name': 컨테이너의 이름 부여
	4. -v 'host-path':'container-path': 볼륨 연결
4. docker rm 'container-id/name': 컨테이너의 id나 name을 이용해 실행중이 아닌 컨테이너 제거
5. docker rm -f 'container-id/name': 실행중인 컨테이너를 제거할때 사용

### docker 관리-거의 사용안함(compose)
1. docker stop 'container-id/name': 컨테이너 멈춤(graceful)
2. docker start 'container-id/name'
3. docker restart 'container-id/name'

### 실행중인 컨테이너와 상호작용
1. docker exec 'container-id/name' 'command': 실행중인 컨테이너 안에서 커맨드 실행
   ex) docker exec -it 'container-id/name' /bin/bash
2. docker logs 'options' 'container-id/name': 컨테이너 로그 보기
	1. --follow: 실시간으로 로그 보기
3. docker attach 'container-id/name': 컨테이너를 터미널에 붙힘
4. docker kill 'options' 'container-id/name': forcing하게 멈춤(deafult option은 SIGKILL)
	1. --signal,-s : 원하는 시그널일때 forcing하게 멈춤
	2. SIGKILL: 즉시 강제 종료

### 컨테이너 디버깅 커맨드
1. doker inspect 'object-id/name': 정보 JSON형태로 리턴
2. docker stats 'container-id/name':실시간 리소스 사용량
3. docker top 'container-id/name': 컨테이너 안에서 실행중인 프로세스
4. docker events: 실시간으로 docker daemon이 보내는 이벤트
