Docker Volume: Docker는 휘발성이기 때문에 컨테이너가 정지되면 모든 데이터가 날아가는 것을 해결하기 위해 고안
- host 디렉토리와는 별개이고 도커에서 관리한다
- 컨테이너의 라이프사이클이 끝나도 지속성있게 남아있다
- 컨테이너 밖에서 데이터 저장 관리 지원
- 여러 컨테이너끼리 공유 가능

## Docker volume의 특징
1. 영속성
2. 데이터 컨테이너간 공유
3. 호스트에 독립적
4. 쉬운 관리

### Docker volume의 종류
1. 익명 볼륨: 이름없이 자동으로 생성되는 볼륨
2. 실명 볼륨: 명시적으로 생성하고 이름이 있는 볼륨
	- 관리와 재사용이 쉽다
- Bind mounts: 호스트(로컬)머신과 도커를 연결한 볼륨

## Docker volume 명령어
1. docker volume creat <volume_name>
2. docker volume ls: volume 목록 보기
3. docker volume inspect <volume_name>
4. docker volume rm <volume_name>
5. docker volume prune: 현시점에서 사용하지 않는(볼륨마운트되어 있지 않은)볼륨을 전체 정리

## 모범 사례
1. 항상 실명 볼륨 사용
2. 바인딩 할때는 민감한 데이터를 조심(회사 정보를 로컬로 바인딩하면 안됨)
3. 익명 볼륨은 사용 자제
4. 규칙적으로 prune를 사용하여 미사용 볼륨을 제거하여 디스크 공간을 확보
