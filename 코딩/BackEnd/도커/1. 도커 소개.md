### 도커
- 오픈 소스 플랫폼
- 빌드,이동,어플리케이션 실행에 도움
- 가벼운 무게,이동 가능,지속적인 관리
- 개발,스테이징,상품이 다른 환경에서 똑같이 적용된다
### 도커의 이점
- 이동성
- 확장성 
	- K8S
	- 수직적 확장
- 속도
	- 즉시 시작
- 효율성
	- 호스트 OS 커널 사용 -> Host os에 의존성있음
	- 가벼운 무게
	- 빠름
- 지속성
	- 어플리케이션 번들링
	- 환경 번들링

## 주요 도커 컴포넌트
1. Docker engine: 컨테이너 관리 프레임워크
	1. 컨테이너를 생성,실행,관리
	2. Server(dockerd)/REST API/CLI
2. Docker Image: Read-only binary file
3. Docker containers: 도커 이미지의 런타임 인스턴스
	1. 컨테이너끼리 커뮤니케이션 가능
4. Dockerfile: 스크립트 파일
5. Docker Compose: 여러개의 컨테이너가 한꺼번에 돌아가면서 시스템을 만드는 도구
6. Docker Hub: Docker image 저장하는 cloud-based registry
	1. 회사마다 레지스트리 만들 수 있음
	2. 개발자들이 pull하거나 push한다

## 사용 예시
1. 개발 환경 세팅 및 테스팅
	1. 고립된 환경에서 구성할 수 있음
2. 마이크로서비스
3. CI/CD(지속적 통합 및 개발)
4. 하이브리드 클라우드 환경과 멀티 클라우드 환경에서 모두 실행 가능
5. 빅데이터와 머신러닝

## Docker work flow
1. Dockerfile만들기
2. Dockerfile을 빌드해서 docker image 만들기
3. 컨테이너로 docker image 실행
4. Docker hub나 다른 레지스트리에 image push
5. image 다운 받아서 운영환경에서 실행
