## 1. RedisTemplate(자료구조 직접 제어)
- 무엇: Redis가 제공하는 기본 자료구조를 명령어 단위로 직접 제어할 수 있는 도구
- 언제: 예시
	- 실시간 랭킹(Sorted Set)
	- 선착순 대기열 처리(List or Set)
	- TTL을 세밀하게 설정하고 조작
- 왜: Reids의 자료구조 특성을 100%활용하여 가장 빠르고 세밀한 데이터 처리 

## 2. Redis Repository(객체 기반 저장)
- 무엇: Spring Data Jpa를 사용하듯이, Java의 객체를 Reids에 통째로 저장하고 조회할 수 있게 해주는 고수준의 추상화 기능
	- 내부적으로 객체를 **Redis의 Hash구조**로 변환하여 저장함
- 언제: 예시
	- 사용자의 인증 토큰을 저장하고 관리할 때
	- 일정 시간 동안만 유지되어야 하는 임시 장바구니 데이터를 저장
	- 인증 번호 등 단기적으로 필요한 사용자 정보 보관
- 왜: Reids의 복잡한 명령어를 몰라도 일반적인 DBMS에 데이터를 넣고 빼는 것과 동일한 방식으로 빠르게 개발 가능

| **구분**               | **제어 방식**       | **주요 대상**        | **개발 초점**               |
| -------------------- | --------------- | ---------------- | ----------------------- |
| **RedisTemplate**    | 세밀한 조작 (명령어 기반) | Redis 자료구조 (자료형) | 성능 최적화 및 특정 자료구조의 특성 활용 |
| **Redis Repository** | 추상화된 조작 (객체 기반) | Java 도메인 객체      | 빠른 개발 속도 및 유지보수 편의성     |
