## 1. spring-boot-starter-data-redis
- 가장 표준 스타터
- Lettuce인 Reids Client를 포함하고 있다
	- Lettuce는 비동기 방식의 Client이다
- 만약 동기방식의 Jedis를 사용하고 싶다면 Lettuce를 제외하고 Jedis의존성을 추가해야 한다

## 2. redisson-spring-boot-starter
- Reids를 단순 캐시가 아닌 분산 락이나 특정 데이터 구조로 정교하게 제어하고 싶을 때 사용
- 분산락(여러개가 하나의 자원에 접근할 때 데이터 일관성 지킴) 구현할 때 Pub/Sub(락이 해제될 때까지 대기하다 알림 수신) 방식 사용
	- 스핀 락(락을 얻을 때까지 무한 루프로 재시도) 방식보다 서버 부하가 적어 동시성 제어에 유리

## 3. spring-session-data-redis
- 세션 관리 효과적으로 처리하기 위한 기술
- 세션 데이터를 서버의 메모리에 저장하는 대신, 외부 스토리지에 저장하고 관리가 가능
- 서버를 여러 대 두는 L4 로드밸런싱 환경에서 특정 서버가 다운되어도 사용자의 로그인 상태가 유지
- 내부적으로 data-redis를 포함하고 있다.
