## Redis Repository 인터페이스
- ReidsRepository interface는 RDBMS와 같은 페이징,정렬과 같은 복잡한 기능이 필요하지 않다.
- Redis Repository는 데이터를 Key-Value형태로 단순하게 저장하고 조회하는 것이 목적이다.
- **따라서, 가장 기본적인 `CrudRepository`면 충분하다**

### Redis Entity
- Redis key-(Field-Value)형태(Redis Hash)를 Java의 Class 형태로 만든다
- TTL을 Entity에서 설정 가능하다
```java
@Getter
@RedisHash(value = "redisHash" timeToLive = 3600)
//value: redis key 접두사
//timeToLive: TTL
public class RedisEntity{
	@Id
	private String id; //실제 키는 위에서 value에 작성한 "redisHash:{} 형태로 생성됨"
	private String token;
	
	//생성자
}
```
