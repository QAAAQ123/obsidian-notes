- 유저 아이디가 중복되면 치명적인 오류가 발생한다

## 중복 방지 로직
- UserEntity의 username 속성에 Unique 제약조건을 설정한다.
- unique = true
```java 
@Column(unique = true)
private String username;
```

## 중복 검증 로직 
1. userRepository에 coustom jpa method로 existsByUsername(String username)을 추가한다.
2. userService에 들어온 DTO와 DB에 들어있는 username이 중복되는지 existsByUsername으로 확인하는 로직을 작성한다.
```java
//repository
boolean existsByUsername(String username);
//service method
//db에 이미 동일한 username을 가진 user가 존재하는지 확인해야함  
boolean isUser = userRepository.existsByUsername(joinDTO.getUsername());  
if(isUser) {  
    return;  
}
```

## 클라이언트 검증 로직
- 클라이언트에서도 회원가입 아이디 중복 확인 로직을 추가해야 한다.
- httpXMLRequest 메소드를 통해 백엔드에 미리 구현해둔 API에 이미 존재하는 username인지 검증하는 로직을 추가해야 한다.
- 하지만 클라이언트에 검증 로직이 있더라도 서버 side에서 다시 한번 중복 검증 로직을 작성해야 한다.

## 가입 불가 문자 정규식 처리
아이디, 비밀번호에 대한 정규식 처리도 필요하다.
가입 불가 정규식 검증을 클라이언트,서버 한 곳에서만 할 수도 있지만, **두 곳에서 모두 검증하는게 가장 권장**된다.

- 아이디의 자리수
- 아이디의 특수문자 포함 불가
- admin과 같은 아이디 사용 불가
- 비밀번호 자리수
- 비밀번호 특수문자 포함 필수
