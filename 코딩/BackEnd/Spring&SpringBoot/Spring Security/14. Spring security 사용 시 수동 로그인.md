Spring Security가 로그인 로직을 수행 할 때는 HMTL 로그인 폼에서 데이터를 서버로 전송할때 사용자이름과 비밀번호를 가로채서 로그인 로직을 수행한다. 
그런데 RestController와 Service에서 수동 로그인 로직을 만들경우 Security의 자동 로그인 로직과 Service의 수동 로그인 로직이 충돌하게 된다. 결국 이 충돌 때문에 로그인이 처리되지 않는 문제가 발생한다. 
결론: Spring security를 사용할때는 수동 로그인 로직을 만들면 안된다. Security Configration에서 로그인을 처리해야 한다.
