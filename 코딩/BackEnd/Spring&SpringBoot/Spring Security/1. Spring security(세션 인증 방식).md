## 간단한 동작 방식
![[spring security 개요도.png]]
- 일반적인 클라이언트의 요청이 들어오면 Tomcat Servlet Container가 요청을 받아서 Controller에 도달한다
1. Spring Security Config파일로 Filter chain을 만든다
2. 인증 되지 않은 사용자(로그인 요청)이 들어오면 filter chain이 요청을 가로채서 인증을 시도한다.
	1. 인증 성공시 인증이 완료된 객체를 생성 및 저장하고, 세션을 만든 후에 Controller에 도달한다
	2. 인증 실패시 실패 응답을 보낸다
3. 인증된 사용자(세션이 있는 사용자)이면 Filter chain에서 사용자가 인증된 상태임을 확인하고 통과하면 Controller에 도달한다.

> Spring security confing 파일로 설정을 하지 않을 시에 모든 경로에대해 security가 적용된다.

## 인가
인가: 특정 사용자가 특정 리소스나 기능에 접근할 수 있는 ==**권한을 가지고 있는지 확인**==하는 과정
- 특정한 경로의 요청이 들어오면 controller에 도달하기 전 필터에서 spring security가 검증함
	1. 해당 경로의 접근은 누구에게 열려 있는지
	2. 로그인이 완료된 사용자인지
	3. 해당되는 role을 가지고 있는지
