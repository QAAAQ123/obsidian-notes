## CSRF
Cross-Site Request Forgery의 약자로 요청을 위조하여 사용자가 원하지 않더라도 서버에 요청을 강제로 보내는 것이다

## 개발 환경에서의 CSRF 설정
- 일반적으로 개발환경에서는 CSRF를 disable한다. 


## 배포 환경에서의 CSRF 설정
- 배포 환경에서는 CSRF 공격을 방지하기 위해 CSRF 설정을 Enable로 변경하고 추가 설정을 한다.
- CsrfFilter를 통해 POST,PUT,DELETE 요청에 대해서 토큰 검증을 진행한다.(**GET은 토큰 검증을 하지 않는다.**)
```XML
<form action="/loginReceiver" method="post" name="loginForm">
	<-- hidden 타입으로 token을 넘겨준다. -->
    <input type="hidden" name="_csrf" value="{{_csrf.token}}"/>
    <input type="submit" value="login"/>
</form>
```
- 원래 logout은 POST 요청으로 진행해야 하지만, 설정을 하면 GET으로도 로그아웃을 진행할 수 있다.
```java
//config 
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception{

    http
            .logout((auth) -> auth.logoutUrl("/logout")
                    .logoutSuccessUrl("/"));

    return http.build();
}
//controller
```
