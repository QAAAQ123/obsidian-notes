- 그냥 객체를 특정 메소드에 넣어서 사용을 하면 테스트에서 비교할 때 잘못 판단할 가능성이 있다(Mockito의 메소드에 따라 동일성,동등성을 비교한다.)
	- 인스턴스 값이 다르면 같은 클래스인데도 다르다고 판단할 가능성 존재
	- 주소 값이 다르면 같은 클래스인데도 다르다고 판단할 가능성 존재
- 따라서 **인스턴스나 주소에 상관없이 비교**할 필요가 있는데, 이 때 사용하는게 Mockito의 any(xxx.class)이다.
- 결론: any는 어떤 xxx.class타입의 객체가 오더라도 일치하다고 판단을 내려주는 유연한 인자이다.
