- 웹 스코프: 웹 스코프는 특정 웹 환경(요청, 세션 등)의 **생명주기에 따라** 빈의 생성과 소멸이 관리되는 것
- ~****={red}모든 스코프는 데이터를 저장하는 역할만 해야 한다. 스코프안에 비즈니스 로직을 담으면 절대 안된다.=~
- 종류
	1. 요청 스코프: 단일 HTTP 요청에서만 존재
	2. 세션 스코프: 전체 HTTP 세션 동안 존재
	3. 애플리케이션 스코프: 앱이 실행되는 동안 존재
- **`@RequestScope`, `@SessionScope`와 같은 스코프 빈은 상태(데이터)를 저장하는 역할만 수행**

## 요청 스코프
- **~={red}HTTP 요청마다 새로운 인스턴스를 생성하는 스코프=~**
- 요청마다 유효성을 검사할 때 사용한다.
- 사용 방법-controller에 의존성 주입하여 사용
```java
@Component
@RequestScope//요청 스코프 형태로 빈에 등록(요청마다 다른 인스턴스 등록)
public class xxxx{
	//데이터
}
```


## 세션 스코프
- **~={red}HTTP 세션이 활성화 되어있는 동안 유지되는 빈의 인스턴스=~**
- 세션의 종료 시점
	1. 세션 만료시간 초과
	2. 클라이언트 종료
	3. 서버 종료
- 사용 방법-controller에 의존성 주입하여 사용
```java
@Component
@SessionScope
public class xxxx{
	//데이터
}
```

## 애플리케이션 스코프
- **~={red}앱이 실행되는 동안 유지되는 빈의 인스턴스=~**
- 프로덕션 앱에서 사용하는 것을 권장하지 않음. 스코프 대신 
- 싱글톤 빈과 비슷하게 동작함
- 사용 방법
```java
@Component
@ApplicationScope
public class xxxx{
	//데이터
}
```
