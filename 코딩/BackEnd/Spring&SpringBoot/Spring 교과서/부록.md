## 모놀리스 방식
모놀리스 방식: 모든 구성 요소가 하나로 구성되어 있는 방식
예시)서점:제품,송장,배송,고객 관리를 하나로 만들어 운영
단점
1. 수평 확장(요청을 분할하여 처리하는 방식)에 적합하지 않음-확장하려면 모든 기능을 확장해야 함
2. 아주 작은 변경 사항이더라도 발생하면 전부 다시 배포해야 함
3. 앱이 모놀리스여서 관리하기 어려운게 아니라 코드의 품질이 안좋아서 관리하기 어려울 수도 있다.

## 서비스 지향 아키텍쳐(SOA)
앱에서 특정 기능이 자원을 많이 소모하는 경우에 자원을 효율적으로 관리하기 위해서 자원을 많이 필요로 하는 기능만 확장하는 방식
- 모든 기능을 하나의 프로세서에서 구현하지 않고 여러 프로세스에서 기능을 구현한다. 

장점
- 많은 자원이 필요한 서비스만 확장하는 것이 가능
- 책임 분리
- 일관성 유지에 유리
- 팀 관리 용이

단점
- 서비스(기능)간 통신을 해야함: 네트워크는 완벽하게 신뢰할 수 있는 것이 아니여서 언제든 기능간 호출이 실패할 수 있음
- 보안: 기능간 네트워크를 이용해 통신하기 때문에 보안에 상대적으로 취약
- 데이터 영속성: DB를 공유할 것인지 공유되지 않은 개별 DB를 상요할 것인지
- 배포: 배포에 복잡성 증가(보안,DB 문제)


## HTTP 형식
### HTTP 요청 형식
1. 요청 URI-필수
2. 요청 메서드-필수
3. 요청 매개변수(쿼리 매개변수)-비필수
4. 요청 HTTP 헤더-비필수
5. 요청 HTTP 본문-비필수
### HTTP 응답 형식
1. 응답 상태-필수
2. 응답 헤더-비필수
3. 응답 본문-비필수

### REST 도구
1. 포스트맨
2. cURL
