### 배울 것
빈 관계 설정 방법:와이어링/오토와이어링/의존성 주입

## 1. 와이어링(wiring)
와이어링: Bean을 생성하는 메서드를 직접 조작하여 Bean을 연결하는 방법
1. Setter를 이용해 Bean간 연결
   ex) car.setTire(tire())
	1. ★★컨텍스트에 이미 tire빈 존재->컨텍스트에 있는 tire 빈 사용★★
	2. ★★컨텍스트에 tire빈이 없을 때->컨텍스트에 tire빈 만든 후에 사용★★
2. 메소드의 매개변수를 이용해 Bean간 연결
   ex) public Car car(Tire tire){... setTire(tire)}

*DI: 프레임워크가 특정 필드 또는 매개변수에 값을 설정하는 기법*


## 2. 오토 와이어링(auto wiring)
오토 와이어링
- 빈을 정의하는 클래스를 변경할 수 있을 때 주로 사용
- @Autowired 어노테이션을 사용하여 Bean을 연결하는 방법
1. 클래스 필드를 이용한 값 주입
   ex) @Autowired Tire tire;
   주입을 받을 클래스에서 값을 변경 할 수 있기 때문에 잘 사용하지 않는다
2. **~={red}생성자로 값 주입=~**
   ```java
   private final Tire tire;
   @Autowired
   public Car(Tire tire){this.tire = tire}
	```

3. setter 이용
   ex)@Autowired
   public void setTire(Tire tire){this.tire = tire}

## 3. 순환 의존성
빈A와 빈B가 서로에게 의존성 주입을 해주어야 하는데, 두 빈 모두 컨텍스트에 없을 때에 순환 의존성이 발생한다.
A를 생성하려면 B가 필요한데 B가 컨텍스트에 없다. 그래서 B를 생성하려 하는데, B를 생성하려면 A가 필요하다. 
ex)public A(B b)와 public B(A a)


## 4. 컨텍스트에 동일한 타입의 빈이 2개 이상 있을때 어느 빈을 주입할지 결정하는 방법
1. 매개변수의 변수명이 컨텍스트의 빈과 이름이 일치하는 경우 -> 매개변수의 변수명과 동일한 빈 선택
```java
//tire1() 존재
@Bean
public Tire tire2(){....}

@Bean
public Car car(Tire tire2){....}
```
2. 매개변수의 변수명이 컨텍스트의 빈과 이름이 일치하지 않는 경우 
	1. **~={red}@Primary를 사용하여 기본 빈 지정 =~**
	   ex)@Primary public Tire tire2(){...}
	2. **~={red}@Qualifier를 사용하여 빈을 주입할 메소드에서 특정 빈 명시적 선택=~**
	   ex)public Car(@Qualifier("tire2") Tire tire){...}
	3. 1,2 어느것도 하지 않으면 예외 발생
