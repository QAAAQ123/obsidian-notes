## 동적 웹 앱 구현
서버사이드 랜더링 흐름도에서 약간의 변형을 하면 동적 웹 앱을 만들 수 있다.
- 컨트롤러에서 랜더링 해야하는 뷰 이름을 반환하고
- 뷰 리졸버에 직접 컨트롤러가 뷰 데이터를 전송해주면 동일한 뷰 요청에 다양한 데이터를 동적으로 표시할 수 있다.
- 컨트롤러->뷰 리졸버->디스패쳐 서블릿 흐름으로 변경한 것이다. 
- 컨트롤러에 모델을 등록하여 동적 웹 앱을 만들 수 있다.
```java
@Controller
public class MainController{

	@RequestMapping("/home")//HTTP 요청에 대한 액션
	public String home(Model page){//모델 타입에 저장된 데이터를 뷰로 전송
		page.addAttribute("username","Katy");
		page.addAttribute("color","red");
		return "home.html";
	}
}
```

## ~={red}★★★HTTP 요청에서 데이터를 얻는 방법★★★=~
1. **HTTP 요청 매개변수(쿼리 매개변수):URI에 매개변수 추가하는 방식**
	1. 전송하는 데이터양이 많지 않을 때
	2. 필수가 아닌 데이터를 보내야 할 때
	3. ~={red}컨트롤러에서 @RequestParam으로 받는다=~
	4. ex)/home?brand=honda&price=7000
2. **경로 변수: 경로 변수 값을 설정하는 방식**
	1. 전송하는 데이터 양이 많지 않을 때
	2. 필수인 데이터를 보내야 할때
	3. 최대 2개의 경로변수 권장
	4. ~={red}컨트롤러에서 @PathVariable로 받는다.=~
	5. ex)/home/brand/price
3. HTTP 요청 헤더
4. HTTP 요청 본문
