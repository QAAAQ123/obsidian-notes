## View의 개념
- ==**View: SELECT 쿼리를 "저장"한 객체(저장된 SELECT 쿼리)**==
- 뷰의 작동 방식
	1. 뷰 조회 요청
	2. 뷰에 저장된 SELECT Query 실행
	3. 테이블에서 데이터 가져옴
	4. Query 결과를 사용자에게 반환
- 뷰를 만드는 방법
```SQL
#뷰 생성
#뷰에서 사용될 열 이름을 AS로 테이블과 다르게 설정 가능
CREATE VIEW 뷰이름
AS
	SELECT 문;

#뷰 사용
SELECT 열_이름 FROM 뷰이름
	[WHERE 조건];

#뷰 수정
#뷰에서 사용될 열 이름을 AS로 테이블과 다르게 설정 가능
ALTER VIEW 뷰이름
AS
	SELECT ....;

#뷰 삭제
DROP VIEW 뷰이름;

#뷰 정보 확인
DESCRIBE 뷰이름;

#뷰 쿼리 확인
SHOW CREATE VIEW 뷰이름;
```

- 뷰를 사용하는 이유
	1. 보안에 도움이 된다.(Select문에 중요 데이터를 선택하지 않게 하고, 사용자에게 role이나 view 권한만 부여하면 되기 때문에)
	2. 뷰를 한번 만들어 놓으면 복잡한 쿼리를 반복 작성의 번거로움을 줄여준다.
- 뷰를 통해 데이터를 수정 할 수 있다.
- 뷰를 통해 데이터를 삽입 할 수도 있지만 권장하지는 않는다
- 조건을 만족하는 뷰를 통해 원본 테이블을 삭제 할 수 있다. 하지만 테이블을 삭제하면 뷰는 더이상 데이터 결과를 보여줄 수 없다
