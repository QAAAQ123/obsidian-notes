1. 일반 예외: 컴파일 과정에서 일어나는 예외/애플리케이션을 실행 할 수 없다
2. 실행 예외: 컴파일 후 예외/애플리케이션 실행 중 발생하며 정지시키지 않는다

## 2. 예외 처리 코드
```java
try {
	
} catch(예외클래스 e){
	예외 처리
} finally{
	항상 실행
}
```
- catch문을 2개 이상 작성하면 예외 발생에 맞게 사상되어 예외를 처리한다

## 3. 리소스 자동 닫기
- try-catch-finally를 사용하여 리소스를 finally단에서 안전하게 닫아도 되지만, try-with-resources를 사용하면 더 간편하게 사용할 수 있다
- **~={red}리소스 자동 닫기를 사용하려면 반드시 AutoColseable interface를 구현하고 close를 Override해야한다=~**
```java
public class ResourceManage implements AutoCloseable{

	//리소스 자동 닫기
	//java8이전 버전은 try안에 리소스 선언해야함
	try(
		FileInputStream file = new FileInputSteam("file.txt");
		FileInputStream file2 = new FileInputSteam("file2.txt")
		){
		//....
	} catch(IOException e){
		//.....
	}
	
	//java9이상부터 외부 리소스 사용 가능
	FileInputStream file = new FileInputSteam("file.txt");
	FileInputStream file2 = new FileInputSteam("file2.txt");
	try(file; file2){
		//....
	} catch(IOException e){
		//....
	}
	
	@Override
	public void close() throws Exception{
		//코드 작성(로그나 sout 등)
	}
}
```


## 4. throws로 예외 떠넘기기
- 메소드를 호출 한 곳으로 예외를 떠넘길 수 있다
- `리턴타입 메소드명(매개변수,....) throws 예외클래스1,예외클래스2 { ... }`
- 클래스에도 throws 사용 가능
- java의 main 메서드에 예외를 떠넘기면 JVM이 예외처리를 한다

## 5. 사용자 정의 예외
- 일반 예외는 Exception의 자식 클래스
- 실행 예외는 RuntimeException의 자식 클래스
```java
public class XXXException extends [Exception | RuntimeException] {
	public XXXException{
		//예외 메시지를 입력 받지 않는 기본 생성자
	}
	
	public XXXException(String message){
		super(message); //예외 메시지를 입력받는 생성자
	}
}
```
