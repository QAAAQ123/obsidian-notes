## Abstract Class
- 사전적 의미의 추상: 실체 간에 공통되는 특성 추출
- ~={red}***추상 클래스는 새로운 실체 클래스를 만들기 위한 부모 클래스로만 사용된다.***=~
- 예시) Bird,Insect,Fish Class는 Animal Abstract Class를 상속 받는다.

### Abstract class 선언
```java
public abstract class 클래스명{
	//field
	//constructor
	//concrete method
	//abstract method
}
```
- 공통된 것만 뽑아서 추상 클래스를 만든다. 
- 추상 클래스에 사용 할 수 있는 것
	1. ~={red}필드=~
	2. ~={red}생성자=~
	3. ~={red}일반 메서드=~
	4. ~={red}추상 메서드=~
- ==**new 연산자를 통해 객체를 만들지 못한다.**==

### Abstract method와 재정의(override)
- Abstract class는 추상 method와 일반(구체적) method 2가지를 가질 수 있다.
- abstract 메소드는 반드시 자식 클래스에서 override하여 구현 해야 한다.
- abstract 메소드는 실행 내용인 중괄호를 가지지 않는다.
```java
public abstract class Animal{
	abstract 리턴타입 메소드명(매개변수,...);
	abstract void sound();
}

public class Dog{
	@Override
	public void sound(){
		sout("멍멍");	
	}
}
```
## Abstract Class 정리
-  ==**공통 특성을 모아놓은 클래스**==
- **추상 클래스는 공통인 특성만 추출해서 담아놓고 상속을 해주는 용도로만 사용된다.평범한 클래스와 달리 추상 메소드를 사용 할 수 있고, 추상 메소드는 반드시 자식 클래스에서 구현 해야 한다.**


## Interface
- 공통점이 없는 자식 클래스에서 가지고 있어야 하는 기능의 정의만 모아 놓고, 구현은 자식 클래스에서 하는 것
- Java에서 다중 상속이 불가능 한 것을 대체한 기능
- 추상적으로 아무런 정의가 되어있지 않지만, 타입으로 쓸 수 있다



## ★★★Abstract Class와 Interface의 이해★★★
![[인터페이스 사용 이유.png]]

- 포유류(개,고양이,고래),조류(독수리,펭귄),어류(상어)로 묶었다
- **고래,펭귄,상어**는 연관성이 없지만 물에서 생활한다는 특징이 있다
- 자바에는 단일 상속만 가능하고 다중 상속은 불가능 하다
- 왜?: 좌측 상단의 그림에서 D는 다중 상속을 받고 있다. 여기서 객체 D를 생성하게 되면 ~={red}메모리에 A객체가 2개가 만들어진다=~.(B와C 모두 A를 상속 받고 있기 때문에)
- A객체가 2개 만들어지면 JVM은 어떤 A를 사용해야 할지 모른다.
- 어떤 A를 사용해야 할지 모르는 것을 **모호함**이라고 한다
<br>
- 그럼 고래,펭귄,상어가 '물에서 생활한다'라는 공통점을 같은 곳에서 상속 받아 올 수 없다. 이 문제를 인터페이스를 통해 해결한다.
- 만약 A가 아무런 정의를 가지고 있지 않고 객체 생성도 불가하다면 두 곳 모두에 문제 없이 상속할 수 있다.(타입은 가진다)
- 그리고 정의를 하지 않은 것들을 상속을 받은 객체에서 해주면 아무 문제 없이 동작한다. 이 것이 인터페이스이다

- 위의 것들을 종합해보면 추상 클래스는 다중 상속 불가,추상 메소드만 구현하면 됨,모든 특성을 자식 클래스에서 사용 할 필요 없음,생성자 사용 가능,**객체 생성 불가**이다.
- 인터페이스는 다중 상속 가능,모든 메소드 자식에서 구현 필수,모든 메소드를 자식이 사용해야함,생성자 가질 수 없음,인터페이스 타입으로 사용가능,**객체 생성 불가**이다.
