## Hash?
- Map between the String fields and String values 
- Object type is a perfect representative hash data type

## Concept of hash in redis
![[Pasted image 20260225115927.png]]
- Reids Key(ASCII)가 Redis Hash를 가리키고 있는 형태이다.
	- Redis Key는 RDBMS의 table 이름과 비슷한 것?
- ~={red}**Redis Key-Reids Hash쌍이 있고 다시 Reids Hash안에 Filed-Value형태로 Hash가 저장되어있는 형태**=~이다
- 40억개 이상의 field-value쌍 저장 가능
	- key-value가 아닌 field-value 쌍이다.


## 명령어
### hmset: Hash mset
- `hmset {redis key} {field1} {value1} {field2} {value2} ...`
- hmset students1 name john age 15 class science

### hget: Hash get
- `hget {redis key} {field}
	- 반드시 2개 인자 모두 입력해야 한다.
	- redis key만 입력하면 어떤 value를 가져올지 모르기 때문

### hgetall: Hash get all
- `hgetall {redis key}`
	- redis key에 해당하는 모든 field-value쌍 보기

### hexists: Hash exists
- `hexists {redis key} {value}`
	- redis key에 value가 있는지 확인
	- return 값은 integer값으로 나옴(0 or 1)

### hedl: Hash delete
- `hdel {redis key} {field1} {field2} ....`

### hsetnx: Hash set if not exists
- `hsetnx {redis key} {field} {value}`
	- 반환값: integer
	- field가 이미 있으면 바꾸지 않고, 없을때만 추가한다

### hkeys: Hash keys
`hkeys {redis key}
- reids key안에 있는 모든 field값들만 return

### hincrby: Hash increase by
`hincrby {redis key} {field} {increasement}`
- integer 타입 value 증가


### hvals: Hash values
`hvals {redis key}`
- redis key에 있는 모든 value값들만 return

### hlen: Hash fields length
`hlen {redis key}`
- reids key에 해당하는 field가 몇개인지 개수 반환

### hmget: Hash mget
`hmget {redis key} {field1} {field2}...`
- 원하는 value들만 선택해서 보여줌
