## Concept of Lists in redis
![[Pasted image 20260225132217.png]]
- Redis list is simply list as a String
	- String들의 배열
- Lists hava a HEAD at the top and a TAIL at the bottom
	- List의 처음에는 HEAD,끝에는 TAIL이 있다
- Can insert data to top and bottom
	- HEAD와 TAIL에 값을 넣을 수 있다.

## Lists structure in Redis
#### Type1(수평)
- `Head <-> Value1 <-> Value2 <-> .... <-> ValueN <-> Tail`
#### Type2(수직)
0. Head
1. Value1
2. Value2
3. Value3
4. ...
5. ValueN
6. Tail


### LPUSH(Left-Push)/RPUSH(Right-Push)
- 수평으로 Lists를 놓으면 왼쪽에 Head가 오고 오른쪽에 Tail이 온다.
- `Left=Head` `Right=Tail`
- 따라서 Left-push는 Head에 element를 넣는 것이고, Right-push는 Tail에 element를 넣는 것이다

## 명령어
### lpush: Left push(Push from the Top)
- 요소를 왼쪽(Head)에 넣는다
	- Top에서 넣으니까 Stack처럼 쌓임
- `lpush {redis key} {value1} {value2} {value3}...`
	- lpush num 1 2 3 4 5의 결과는 1 2 3 4 5가 아닌 5 4 3 2 1순서이다

### lrange: Left range
- `lrange {redis key} {start index} {stop index}`
	- 시작부터 끝 인덱스의 value들을 보여준다
- `lrange {redis key} 0 -1`: Lists의 모든 value보기

### lpop: Left pop(Left remove)
`lpop {redis key}`
- Head에서 Pop함

### rpush: Right push(Push from the Bottom)
 - 요소를 오른쪽(Tail)에 넣는다
	- Bottom에서 넣으니까 가장 마지막 index에 들어감
- `rpush {redis key} {value1} {value2} {value3}...`
	- rpush num 1 2 3 4 5

### rpop: Right pop(Right remove)
- `rpop {redis key}`
- Tail에서 Pop함

### llen: Left length
- `llen {redis key}`

### lindex: Left index
`lindex {redis key} {index}`
 - index에 있는 value보기
 - Head에 가까운 값부터 index 0이 붙는다
 - `Head<->index0<->index1....<->Tail`

### lset: Left set
- `lset {redis key} {Left Index} {value}`
- Head부터 인덱싱 한 값을 기초로 원하는 index에 값을 넣는다
	- lset name 3 john: Head로부터 4번째에 있는 곳에 john을 넣는다(덮어씌운다)


### lpushx: Left push if the key exist
`lpushx {redis key} {value}`: 이미 redis key에 대한 List가 생성되어 있을 때에만 value를 left-push한다
- return type: integer

### linsert: Left insert
`linsert {redis key} BEFORE|AFTER {pivot} {value}`
- `pivot`: value를 넣을 기준이 되는 값
- linsert num before 2 55
	- key가 num인 List에 2라는 값이 있으면 그 전 인덱스에 55를 넣는다
