
### 🚀 한눈에 보는 요약표

|**메서드**|**역할 (CRUD)**|**SQL 비유**|**특징**|**실무 예시**|
|---|---|---|---|---|
|**GET**|**조회 (Read)**|`SELECT`|데이터 변형 없음. 주소창(URL)에 파라미터 보임.|사상의학 질문지 가져오기|
|**POST**|**생성 (Create)**|`INSERT`|새로운 데이터 만듦. Body에 데이터 숨겨 보냄.|회원가입, 사상의학 결과 제출|
|**PUT**|**수정 (Update)**|`UPDATE`|데이터 **전체**를 덮어씀 (갈아끼우기).|회원 정보 전체 수정|
|**DELETE**|**삭제 (Delete)**|`DELETE`|데이터 삭제.|회원 탈퇴|

### 📝 상황별 상세 가이드

#### 1. GET (주세요)

- **언제 써?** 서버한테 "나 이거 보여줘", "검색해줘" 할 때.
    
- **핵심:** 서버의 데이터가 **절대 바뀌면 안 됩니다.** (단순 조회용)
    
- **예시:**
    
    - `GET /api/sasang/test` (질문 목록 내놔)
        
    - `GET /api/onboarding/check?nickname=불주먹` (이 닉네임 있어? 확인만 해줘)
        

#### 2. POST (처리/저장해주세요)

- **언제 써?** "새로운 걸 등록"하거나, "로그인"처럼 중요한 데이터를 보낼 때.
    
- **핵심:** 실행할 때마다 새로운 데이터가 계속 생깁니다. (글쓰기 버튼 두 번 누르면 글 두 개 됨)
    
- **예시:**
    
    - `POST /api/onboarding/complete` (내 정보 저장해줘 → 회원 생성)
        
    - `POST /api/login` (아이디/비번 줄게 로그인 처리해줘)
        

#### 3. PUT (이걸로 다 바꿔주세요)

- **언제 써?** 기존 데이터를 **통째로** 교체할 때.
    
- **핵심:** **"덮어쓰기"**입니다. 만약 이름, 나이, MBTI가 있는 회원정보에 `PUT`으로 "이름"만 보내면, 나이와 MBTI는 지워(Null)질 수 있습니다. (전체를 다 보내야 함)
    
- **예시:**
    
    - `PUT /api/members/me` (내 정보 이걸로 싹 다 바꿔줘)
        

#### 4. DELETE (지워주세요)

- **언제 써?** 데이터를 없앨 때.
    
- **핵심:** 삭제 후에는 다시 조회해도 없어야 합니다.
    
- **예시:**
    
    - `DELETE /api/members/me` (나 탈퇴할래)

### 💡 꿀팁: PUT vs PATCH (면접 단골 질문)

실무에서는 **PUT**보다 **PATCH**를 더 많이 쓰기도 합니다.

- **PUT (전체 수정):** "파일 덮어쓰기" 느낌. (안 보낸 건 사라짐)
    
- **PATCH (부분 수정):** "파일 내용 고치기" 느낌. (보낸 것만 바뀜)
    

> **상황:** 사용자가 닉네임만 바꾸고 싶어 함.
> 
> - `PUT`을 쓰려면: 닉네임 + 원래 있던 나이 + MBTI... **다 다시 보내야 함.**
>     
> - `PATCH`를 쓰면: **{ "nickname": "새이름" }** 이것만 보내면 됨. (편함 👍)
>     

**지금 만드시는 온보딩 완료(`complete`)는 "없던 회원을 새로 등록(완성)"하는 것이니 `POST`가 맞습니다!**
