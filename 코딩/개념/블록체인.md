## 개념
- 참여자가 정보를 공유한다
- 특정 사용자가 위조하더라도 나머지 사람들의 정보를 통해 확인 가능
- 하나만 남아있더라도 모든 정보 복구 가능

## Block + Chain
### Block
- 특정 시간동안 쌓인 거래 기록이 저장되는 장소
- 특정시간동안의 거래 기록이 블록에 담길때 전체 50%이상의 사람들과 일치하는지 확인되면 블록 완성
- 블록 완성되면 그전의 블록과 연결되는 과정 반복
- 거래 기록 -> 일치하는지 확인 ->블록 완성 -> 그전 블록과 연결

### Chain
- 블록이 연결되어 있다
- 같은 정보를 공유하면서 다시 연결됨
- 새로운 사용자는 모든 블록을 다운 받아서 참여하게 됨

### 블록체인 보안 원리
- 블록체인에 참여한 사람은 모든 사람의 정보를 저장하고 있지만 **자신의 정보밖에 볼 수 없다**
- 해시를 이용해 강력한 보안
- 기존에 있던 블록을 위,변조하거나 새로운 블럭을 위,변조 할 경우 모두 해쉬 값을 통해 보안 함

## Hash
- 원본 데이터를 고정된 길이의 데이터로 변환
- 특징
	- 하나의 데이터는 하나의 해시값 가짐(고유한 값)
	- 항상 고정된 크기로 변환됨(원본의 크기와 상관없음)
	- 아주 조금이라도 변하면 해쉬값 완전히 달라짐
- 블록간의 연결 담당
	- 각 블록은 아래 3개 해쉬 값 가지고 있음
	1. 이전 블록 해쉬값
	2. 10분동안 거래 내역
	3. 해당 블록 해쉬값 

### 거래상황 예시
1. A와 B 검증 과정-서로 같은 블록을 가지고 있으면 됨->신뢰도 보장
2. A와 B 상호간 해쉬값 보여줌/다른 사용자들도 해쉬값 검증에 참여
3. 일치하면 신뢰할 수 있게 된다. -> 거래 시작
4. 해쉬를 통해 효율적이고 확실한 보안 시스템 구축
