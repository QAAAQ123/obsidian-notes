### 1. HTTP 헤더의 휘발성

프론트엔드(React)에서 백엔드로 요청을 보낼 때 헤더에 `Authorization: Bearer <게스트 토큰>`을 실어 보냈더라도, 백엔드가 구글 로그인 페이지로 **리다이렉트(302 응답)**를 시키는 순간 브라우저는 **새로운 요청**을 생성하여 구글 서버로 날아갑니다. 이때 기존에 프론트엔드 메모리나 요청 헤더에 있던 토큰 값은 구글로 전달되지 않고 증발합니다.

### 2. 구글에서 돌아올 때의 상태

사용자가 구글 로그인을 마치고 우리 서비스의 `OAuth2SuccessHandler`로 돌아올 때, 구글은 우리 서버에 **'이 사용자의 인증 정보'**는 넘겨주지만, **'이 사람이 이전에 어떤 게스트였는지'**에 대한 정보는 알지 못하며 넘겨주지도 않습니다.
