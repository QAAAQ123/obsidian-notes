## Rest API 방식
#### 정의
특정 언어나 라이브러리에 종속되지 않고, HTTP 프로토콜(URL 이동)을 통해 인증서버와 직접 통신하는 방식
#### 동작 원리
페이지를 이탈함
1. 브라우저의 주소창을 물리적으로 이동(`href`)시켜 로그인 페이지를 띄움
2. 로그인이 끝나면 Code와 함께 Redirect URI로 설정해둔 곳으로 이동함
3. 이 과정에서 반드시 페이지 새로고침이 발생


## SDK 방식
#### 정의
플랫폼 제공자(구글, 카카오)가 로그인을 쉽게 구현하라고 만들어둔
'전용 자바스크립트 라이브러리'를 이용하는 방식.

#### 특징 (핵심)
개발자가 '팝업'과 '리다이렉트' 중 원하는 방식을 선택할 수 있음.

#### 동작 원리 (두 가지 모드)
1. 팝업 모드 (Kakao.Auth.login)
     - 팝업창을 띄워 로그인.
     - 현재 페이지 유지됨. (게스트 정보 변수에 유지 가능)
     - Callback 함수로 Code 받음
2. 리다이렉트 모드 (Kakao.Auth.authorize)
     - 페이지를 이동시켜 로그인. (새로고침 발생)
     - 현재 페이지 떠남. (게스트 정보 LocalStorage 저장 필수)
     - 돌아온 URL 뒤에 ?code= 로 받음.
