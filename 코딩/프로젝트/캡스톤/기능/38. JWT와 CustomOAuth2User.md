- CustomOAuth2User는 SecurityContextHolder에 등록하기 위한 사용자 객체이다
- CustomOAuth2User에 필요한 정보를 JWT가 가지고 있기 때문에 JWT를 분해한다.
- JWT 자체는 인증만을 위한 것이다.
- 인가는 ROLE이나 다른 형태로 한다.
---
### 1. 토큰 (JWT): 요청 인증 데이터

클라이언트(프론트엔드)가 서버에 자신이 누구인지 증명하기 위해 HTTP 요청 헤더에 담아 보내는 데이터 형식입니다. 서버는 이 토큰의 서명과 만료 기간을 검증하여 해당 요청을 처리할지 거부할지 판단합니다.

### 2. CustomOAuth2User: 서버 내부 사용자 객체

서버 내부에서 현재 요청을 보낸 사용자의 정보(이메일, 권한 등)를 사용하기 위해 정의한 클래스입니다. 이 객체는 `SecurityContextHolder`에 등록되어, 컨트롤러나 서비스 계층 어디서든 `@AuthenticationPrincipal` 어노테이션을 통해 즉시 접근할 수 있습니다.

### 3. 데이터 흐름 (토큰 분해 및 객체 생성)

스프링 시큐리티의 `JWTFilter`가 이 둘 사이를 연결합니다.

1. 클라이언트가 토큰을 보냅니다.
    
2. `JWTFilter`가 토큰을 해독(파싱)하여 내부의 데이터(Claim)를 꺼냅니다.
    
3. 꺼낸 데이터(email, role 등)를 `MemberDto`에 담고, 이를 `CustomOAuth2User` 객체로 생성합니다.
    
4. 생성된 객체를 `SecurityContextHolder`에 등록합니다.
    

결론적으로, **토큰은 클라이언트와 서버 간의 인증 통신 규격**이고, **CustomOAuth2User는 서버 내부 요청 처리 과정에서 메모리에 들고 있는 사용자 데이터 객체**입니다.
