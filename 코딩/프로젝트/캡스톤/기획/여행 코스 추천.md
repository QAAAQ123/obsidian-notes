## 1. 기능 개요
#### 목적
- 사용자의 현재 위치를 기반으로 채팅창에 여행 코스 추천을 요청하면, AI가 경로들을 지도에 제시하고 사용자가 이를 간편하게 수정하여 최종 상세 여행 코스를 생성하도록 지원합니다.

### 타겟 사용자
- 인천에 대한 사전 정보 없이 현지에 도착하여 즉흥적으로 여행을 즐기고자 하는 관광객을 대상으로 합니다. => 결정 피로도 감소
- 이미 여행 계획을 모두 수립하고 방문한 사용자는 고려 대상에서 제외하며, 현장에서의 실시간 코스 제안이 필요한 사용자에게 집중합니다.

## 2. 핵심 플로우
1. **요청 단계**: 채팅창 화면에서 "한국의 가을을 느낄 수 있는 코스를 추천해줘"입력
2. **경로 탐색 및 제시 단계**: 화면에 위치 마커들이 생성되며, 추천 경로 옵션들(예: 경로A,경로B,경로C)이 지도 위에 선으로 표시됨
3. **경로 수정 팝업 표시 단계**: 화면에 마커들이 생성될 때, 경로를 수정 할 수 있는 팝업이 자동으로 생성됨
4. **경로 수정 단계**: 지도 측면의 작은 팝업을 통해 제시된 경로 내의 장소를 삭제하거나,AI가 제안하는 추가 장소를 코스에 추가함
5. **경로 확정 단계**: 수정 팝업에서 `완료/저장`버튼 클릭
6. **상세 코스 생성 단계**: 하단의 '여행 코스'탭으로 자동 이동되며, 확정된 장소들의 썸네일,주소,정보 등이 포함된 상세 리스트가 생성됨

- `여행 코스 탭`: 여행 코스 생성 이전의 `여행 코스`탭 에는 '코스 생성하기' 버튼이 있고, 버튼을 누르면 채팅 화면으로 강제 리다이렉트

## 3. 화면별 세부 요구사항
- **지도 및 채팅 화면**
	- 경로 편집 팝업
	- 경로를 시각적으로 보여주는 지도 위의 선과 번호가 매겨진 마커
	- 경로 생성 시간동안 화면 처리(예: `코스를 생성하고 있습니다.`)
- **경로 편집 팝업**
	- 현재 선택된 경로 순서가 있는 리스트
	- 각 장소 옆에 삭제`-`버튼
	- 하단 다른 추천 장소에 `+`버튼
	- `경로 확정하기`버튼
- **여행 코스 탭**
	- 코스 재수정 기능
	- UI 리스트 구성 요소
		- 장소명
		- 썸네일 이미지
		- 주소
		- 상세 정보
		- 외부 연결 링크

## 4. 미결정 사항
- 이동수단 확인 방법
	- 도보와 대중교통
- 코스 유지 기간-x시간,24시에 초기화 등..
- 경로 편집 화면 상세 정보 확인 
	- `자세히 보기`-터치하면 해당 칸이 길게 늘어나면서 정보 나타남
	- 마커 터치하거나 편집 리스트의 장소 터치하면 '코스 편집' 슬라이드 전체가 한개의 장소의 정보만 보여줌(`< 뒤로 가기`)
	- 마커 터치시 지도에 네모 팝업창으로 상세 정보 표시
	- 장소명 옆 '상세 정보' 터치 유지시 상세 정보 보여줌
- 2.2 경로 탐색 및 제시 단계
	- 생성할 추천 경로의 개수(예: 코스A,코스B...)
	- 추천할 장소의 개수(예: 관광지x개,음식점x개,카페x개...)
	- 추천할 장소의 구성(예: 관광지->음식점->카페->관광지 or 관광지->카페->관광지->음식점)
		- 시간대에 따라 다르게 생성 할 수 있으면 좋을 것 같음
			- 12-13시이면 음식점부터
			- 8-10시이면 관광지부터
- 2.3 경로 수정 단계
	- 대체 추천 장소의 개수
	- 대체 추천 장소의 구성
	- 장소 추가,삭제시 동선 재정렬 방식
- 3.2 경로 편집 팝업
	- 경로 편집 팝업의 위치와 형태
		- 팝업 or 슬라이드(채팅창처럼)
		- 상하좌우 위치
- 3.2 여행코스 탭
	- 상세 정보에 들어갈 정보 목록(주소,요금,현재위치에서 거리...)
- 최대,최소 장소의 수
	- 최대 음식점,관광지,카페 허용 개수
	- 최소 음식점,관광지,카페 허용 개수


## 예외 상황
1. 사용자의 위치를 찾을 수 없을 때
2. 주변에 추천할 장소가 없을 때
3. 사용자가 모든 장소를 삭제하려고 할 때
4. 경로 생성 실패 및 서버 오류
