#staticMethod #staticVariable
## static variable
==데이터가 딱 하나만 생성되어 모든 객체가 공유하는 변수==
- 프로그램이 시작될 때 메모리에 딱 한 번 올라가고, 프로그램이 끝날 때까지 사라지지 않는다
- 사용 
	- 모든 객체가 공통으로 가져야 하는 값
	- 전역적으로 공유해야 하는 값
- 주의점: 메모리에 계속 남아 있기 때문에, List같은 큰 데이터를 담아두고 계속 추가하면 메모리 누수의 주범이 된다.
- static variable은 별도의 고정된 메모리 영역에 올라간다. 이 영역은 Garbage Collector의 대상이 아니다.

## static method
==객체를 생성하지 않고도 바로 호출할 수 있는 메서드==
- 인스턴스 변수를 사용하지 않는, 순수 기능 위주의 메서드
- 사용
	- 유틸리티 함수
	- DTO변환 메서드
- 메모리 영향: 메서드의 코드만 메모리에 있고, 실행 시 내부 변수는 스택에 잠시 생겼다가 사라지므로 메모리 걱정없이 써도 된다. 

## 코드만 메모리에 올라가 있다
- 자바는 클래스의 바이트코드만 **Method Area**에 저장함(읽기 전용)
- static method를 호출하는 순간 Method Area에 있는 method를 보고 stack을 생성함
- method가 return을 해서 끝나면 stack에서 사라진다
