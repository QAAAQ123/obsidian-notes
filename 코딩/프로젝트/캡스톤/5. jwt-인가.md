## JWT Filter -> filter로 토큰 없는 사용자 걸러내기
- ==Token기반 인증은 stateless방식이기 때문에 HTTP request가 들어올 때마다 JWT를 생성해줘야 한다.==
1. authorization 값 꺼냄
2. authorization이 없거나 Bearer로 시작 안하면 넘어감
3. Bearer이 없는 순수 토큰 값 꺼냄
4. 순수 토큰 만료여부 확인-> 만료면 넘어감
5. 순수 토큰에서 email,role획득
6. email,role로 MemberDTO 생성
7. authToken 생성-> 사용자 정보 담은 authorization 토큰 생성됨
	1. UsernamePasswordAuthenticationToken(유저,인증수단(null),권한 목록)
8. 세션에 authToken 등록
