### Preflight란?
- pre + flight: 미리 요청을 보내서 안전한지 확인
- 사전 점검 과정
- OPTIONS 메서드를 보냄

## 언제 발생
- 단순 요청이 아닌 복잡하거나 민감한 요청일 때 발생
- 예시
	- Authorization: 토큰 
	- application/json으로 데이터 보낼때
	- 커스텀 헤더를 붙혔을때 

## 동작 흐름
1. 브라우저에서 실제 데이터(Get,Post등)을 보내기 전에 먼저 `OPTIONS`라는 빈 껍데기 요청을 서버에 보냄
2. 서버는 CORS 설정을 확인하고 답을 해줌
	1. 허용(200): 헤더와 메서드 다 허용
	2. 거부(403): 인증 안 되어서 거부
3. 브라우저에서
	1. 허락받으면 진짜 요청을 보냄
	2. 거절당하면 CORS에러를 띄우고 통신을 끊음

## 403 에러가 Ngrok OPTIONS요청에서 났던 이유
- Preflight 요청(OPTIONS)요청은 빈 껍데기라서 토큰 없이 보냄
- 그런데 SecurityConfig는 로그인과 Swagger만 빼고 모두 토큰이 있어야 통과하게 설정해 놓아서 403
- 따라서 OPTIONS 요청은 무조건 통과시키게 설정을 바꿔줬음
