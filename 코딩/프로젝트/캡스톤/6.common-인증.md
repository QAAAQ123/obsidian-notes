## dto-CustomOAuth2User(OAuth2User 구현)
- provider마다 속성 이름이 다름
- provider마다 다른 속성 이름을 일성있게 바꿈

## handler->dto로 refresh,accessToken만들어서 저장,전송
- dto에 들어간 email과 role 추출
- accessToken과 refreshToken 생성 
- redis와 Http 쿠키에 refreshToken 저장
- accessToken은 redirectUrl과 accessToken을 결합하여 프론트로 넘겨줌

## Service->User 정보 받아서 표준화한DTO 형태로 데이터 변형
- OAuth2 Provider의 사용자 정보를 가져옴
- 사용자 정보 추출
- 사용자 정보를 DB 저장또는 업데이트
- OAuth2User로 변환하여 리턴함
