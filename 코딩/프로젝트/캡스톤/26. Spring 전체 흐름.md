### ➡️ 요청 진입 (Incoming Request)

**1. Client (Browser/App)-Http Request**
- 사용자의 동작에 의해 HTTP 요청 패킷을 만들어 서버로 전송합니다.

**2. WAS (Tomcat)**
- 네트워크 신호를 받아 자바가 이해할 수 있는 `HttpServletRequest`, `HttpServletResponse` 객체를 생성합니다.

**3. Spring Security Filter Chain**
- DispatcherServlet에 닿기 전, JWT 인증, CORS, 권한 체크 등 보안 검문을 먼저 수행합니다.

**4. DispatcherServlet**
- 스프링 MVC의 입구로서, 요청 URL을 보고 어떤 컨트롤러가 처리할지 결정합니다 (Routing).

**5. HandlerInterceptor**
- 컨트롤러가 실행되기 직전에 가로채어 로깅이나 추가적인 전처리를 수행합니다 (`preHandle`).

**6. Controller**
- 실제 비즈니스 로직을 실행하고, 결과 값(DTO/Entity)을 `ResponseEntity` 등으로 반환합니다.

### ⬅️ 응답 반환 (Outgoing Response)

**7. HttpMessageConverter**
- 컨트롤러가 반환한 자바 객체(Java Object)를 클라이언트가 이해할 수 있는 JSON 문자열로 변환합니다.

**8. HandlerInterceptor**
- 컨트롤러 처리가 끝난 후 후처리를 수행합니다 (`postHandle`, `afterCompletion`).

**9. DispatcherServlet**
- 요청 처리가 완료되었음을 확인하고 응답 객체 생성을 마무리합니다.

**10. Spring Security Filter Chain (역순)**
- 들어올 때 거쳤던 필터들을 역순으로 빠져나오며(Return) 최종 마무리 작업을 합니다.

**11. WAS (Tomcat)**
- 완성된 `HttpServletResponse` 객체를 HTTP 프로토콜(텍스트/바이트)로 변환하여 네트워크로 전송합니다.

**12. Client (Browser/App)-Http Response**
- 응답받은 JSON 데이터를 해석하여 사용자 화면(UI)을 갱신합니다.
