# [기획서] AI 기반 공시 정성 분석 및 자동 가치평가 시스템

## 1. 개요 (Overview)

- **프로젝트명:** AI 기반 심층 공시 분석 및 자동 가치평가 어시스턴트
    
- **기획 배경:**
    
    - 본업(취업 준비 및 직장 생활)으로 인해 투자 분석에 할애할 시간 부족.
    - 10-K, 10-Q 등 방대한 공시 자료의 핵심을 파악하는 데 과도한 리소스 소요.
    - 기존 요약 서비스는 단순 정보 압축에 그쳐, 리스크의 구조나 전략의 질적 수준을 충분히 분석하지 못함.
        
- **목적:**
    
    - 공시 문서를 구조화하여 정성적 리스크를 체계적으로 분석.
    - 경영진 서술의 구체성, 일관성, 대응 전략 수준을 정량·정성적으로 평가.
    - DCF 기반 가치평가와 결합하여 투자 판단에 활용 가능한 통합 리포트 생성.
        

---

## 2. 핵심 분석 방법론 (Core Methodology)

본 시스템의 차별점은 **정성 데이터를 구조화 가능한 분석 프레임워크로 변환하는 것**입니다.

단순 요약이 아니라, 공시 문장의 **언어 패턴·구체성·리스크 명시 수준·전략 차별성**을 평가합니다.

### 2.1 정성 분석 프레임워크

| 분석 대상 (Items) | 분석 관점 | AI 분석 지침 | 사용자 출력 |
|------------------|----------|--------------|-------------|
| **Risk Factors (Item 1A)** | 리스크 구체성 및 책임 명확성 평가 | 수동적·모호한 표현 탐지, 반복적 일반론 여부 분석, 정량적 수치 부재 여부 확인 | "구체성 결여된 리스크", "책임 소재 불명확", "반복적 일반 리스크 서술" |
| **MD&A (Item 7)** | 문제 인식 수준 및 대응 전략 평가 | 구체적 위협과 추상적 불확실성 구분, 대응 방안의 실행 가능성 분석 | "구체적 대응 전략 부족", "위협 요인 명확", "시장 불확실성 과도 강조" |
| **Business/Strategy (Item 1)** | 전략 차별성 및 경쟁 우위 분석 | 경쟁사 대비 차별성, 트렌드 추종 여부, 진입장벽·해자 구조 존재 여부 평가 | "차별화 전략 미흡", "트렌드 의존형 구조", "경쟁우위 구축 시도 확인" |

## 3. 단계별 구축 계획 (Roadmap)

### [1단계] 10-K/10-Q 텍스트 분석 시스템 (MVP)

- **목표:** 주요 공시의 Item별 구조화된 분석 리포트 자동 생성.
    
- **기능 명세:**
    
    - **Input:** Sec.gov에 올려져있는 공시 htm,XBRL 형식의 문서
        
    - **Processing:**
        - SEC 공시 원문 파싱 및 HTML 정제
        - 주요 섹션(Item 1, 1A, 7, 8) 자동 분리 (Chunking)
        - LLM API 호출을 통한 정성 분석 프레임워크 적용
        - 언어 패턴 기반 리스크 분류 및 전략 평가
        
    - **Output:** 섹션별 핵심 분석 리포트 (구조화된 인사이트 중심)
        

---

### [2단계] 정량적 분석 및 DCF 모델 자동화

- **목표:** 재무 데이터 기반 기업 가치 산출
    
- **기능 명세:**
    
    - **Data Gathering:** 재무제표(BS, IS, CF) API 연동
        
    - **Calculation Logic (Python):**
        
        - **WACC:** 무위험이자율, Beta, 세후 차입비용 반영 자동 계산
        - **FCFF:** NOPAT, CAPEX, 운전자본 변동 포함 잉여현금흐름 계산
        
    - **Scenario Planning:**
        
        - 1단계 정성 분석 결과를 반영하여 성장률 및 마진 가정 설정
        - **Bull:** 전략 구체성 높고 경쟁 우위 확인 시
        - **Base:** 산업 평균 수준 반영
        - **Bear:** 리스크 구체적이고 대응 전략 부족 시
        
    - **Output:** 시나리오별 적정 주가, 현재가 대비 괴리율, 가정 설정 근거 
        

---

### [3단계] 8-K 이벤트 분석 시스템

- **목표:** 수시공시 발생 시 영향도 자동 평가
    
- **기능 명세:**
    
    - 8-K 공시 자동 감지
    - 이벤트 유형 분류 (경영진 교체, 실적 수정, 법적 분쟁 등)
    - 재무 영향 단기/장기 구분
    - 단발성 이벤트 vs 구조적 리스크 판단
        
    - **Output:** 3줄 요약 + 재무적 영향 분석
        

---

### [4단계] 통합 자동화 시스템

- **목표:** 티커 입력 한 번으로 전체 분석 자동 수행
- **Pipeline:**
    1. 티커 입력
    2. 최신 10-K, 10-Q 및 최근 8-K 자동 수집
    3. 재무 데이터 수집 및 DCF 계산
    4. 정성 분석 수행
    5. 종합 리포트 생성 (정성 + 정량 통합)
