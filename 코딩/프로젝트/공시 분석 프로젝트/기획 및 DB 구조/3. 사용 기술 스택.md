
## 🛠 기술 스택 (Tech Stack)

### 🌐 Web Framework & Server

- **FastAPI**: 고성능 비동기 웹 프레임워크를 사용하여 효율적인 데이터 처리 구현
    
- **Uvicorn [standard]**: ASGI 서버 구현체로, 프로덕션 환경의 성능 최적화
    
- **Jinja2**: 서버 사이드 템플릿 엔진을 통한 동적 웹 페이지 렌더링
    
- **python-multipart**: HTML Form 데이터 및 파일 업로드 처리 라이브러리
    
- **Pydantic-settings**: 환경 변수(.env)를 Pydantic 모델로 구조화하여 관리
    

### 🤖 AI & Data Collection

- **Google-generativeai**: Google Gemini API SDK를 활용한 공시 분석 에이전트 구축
    
- **HTTPX**: 비동기 HTTP 클라이언트를 사용하여 SEC 서버와의 효율적인 통신 구현
    
- **BeautifulSoup4**: 수집된 HTML 문서의 데이터 파싱 및 구조화
    
- **lxml**: BeautifulSoup의 성능 극대화를 위한 고성능 C 기반 XML/HTML 파서
    

### 📂 Database & Storage

- **SQLAlchemy**: Python 객체 기반의 ORM(Object-Relational Mapping)으로 데이터베이스 조작 및 설계
    
- **aiosqlite**: SQLite를 위한 비동기 데이터베이스 드라이버로 I/O 병목 현상 방지
    

### ⚙️ Utilities & Automation

- **APScheduler**: 주기적으로 SEC 공시를 확인하고 분석 파이프라인을 트리거하는 작업 스케줄러
    
- **loguru**: 비동기 환경에 최적화된 로깅 라이브러리로, 시스템 상태 모니터링 및 디버깅 효율화
    
- **Python-dotenv**: 보안이 필요한 API Key 및 설정값 관리를 위한 환경 변수 로드
    

---

## 💡 기술 스택 선정 이유 (면접 대비용 Tip)

1. **고성능 데이터 파이프라인**: `APScheduler`로 주기적인 작업을 관리하고, `HTTPX`와 `FastAPI`의 비동기 처리를 통해 수십 개 기업의 대용량 공시(10-K)를 **병렬(Parallel)로 수집·분석**하여 전체 데이터 처리 시간(Throughput)을 획기적으로 단축했습니다.
    
2. **비동기 I/O 최적화 및 안정성**: DB 저장(`aiosqlite`)과 외부 API 통신(`HTTPX`) 간의 병목을 비동기로 해결했습니다. 특히 `loguru`를 도입하여 비동기 루프 내에서 발생하는 이벤트를 가시화하고, 예외 상황 발생 시 추적 가능성을 높였습니다.
    
3. **데이터 무결성 및 확장성**: `Pydantic`으로 비정형 공시 데이터의 스키마를 엄격히 정의하여 데이터 오염을 방지하고, `lxml`을 선택하여 수십만 자에 달하는 복잡한 HTML 파싱 시의 메모리 효율성을 확보했습니다.
