#비동기 
### FastAPI의 비동기
- FastAPI의 비동기는 Single-Thread 방식의 비동기를 주로 사용한다

## await(실행 일시 정지 및 제어권 반환)
- async 함수 내에서 비동기 작업(Awaitable 객체)의 완료를 기다리는 키워드
- 작동 원리
	1. await를 만나면, 현재 함수 A의 실행 중단
	2. CPU의 제어권을 이벤트 루프에 반환(이때 이벤트 루프는 대기 중인 다른 작업 처리)
	3. 기다리던 작업(I/O,DB조회)이 완료되면, 이벤트 루프가 다시 A함수로 돌아와 중단된 지점에서 실행 재개

## yield(값 산출 및 상태 보존 대기)
- 함수를 종료(return)하지 않고, 값을 호출자에게 전달한 뒤 실행 상태를 유지한채 대기
- 작동 원리 (FastAPI 의존성 주입 기준) 
	1. A함수 실행 중 yield 변수명을 만나면, 해당 변수(객체)를 호출자에게 전달. 
	2. 현재 함수의 실행 상태(변수 값, 메모리 컨텍스트)를 그대로 보존한 채 **A함수의 실행을 일시 정지**
	3. 제어권이 호출자로 넘어가 호출자의 로직을 모두 수행. 
	4. 호출자의 작업이 끝나면, 다시 함수A의 yield 다음 줄로 제어권이 돌아옴. 
	5. yield 다음 줄 부터 코드를 실행하고 함수A 종료.

## 네트워크 비동기 요청(Httpx,Fetch)
- 비동기 방식으로 HTTP 요청을 보내기 위해 사용
- Python은 내장 함수가 없어서 Httpx 외부 라이브러리를 사용하고
- JavaScript는 내장 함수가 있어서 fetch를 사용한다
- 작동 원리
	1. httpx나 fetch를 사용하여 Http 요청
	2. 요청을 보내자마자 응답을 기다리지 않고 즉시 Coroutine/Promise를 반환
	3. await 키워드를 작성 해야만 실제 응답 데이터를 받을 수 있음

## 약속 객체(Coroutine,Promise)
- 비동기 작업의 상태와 미래의 결과값을 담고 있는 객체
- Python은 Coroutine또는 Task라고 부르고
- JavaScript는 Promise라고 부른다
- 작동 원리
	1. 비동기 함수를 호출하면, 작업이 완료되지 않았더라도 즉시 Coroutine/Promise를 반환
	2. 이 객체는 현재 대기중(Pending) 상태이며, 내부에 실제 데이터는 없음
	3. await 키워드를 붙히면 이 객체가 완료(Done) 상태가 될 때까지 기다림
	4. 작업이 성공하면 객체 내부의 결과값을 꺼내서 반환하고, 실패하면 에러를 던짐

## 동시 실행
- 여러 작업을 아주 짧은 시간 단위로 번갈아 가며 수행하여 동시에 진행되는 것처럼 보이게 하는 기술
- Python의 asyncio(FastAPI) 환경에서는 주로 싱글 스레드 기반의 동시 실행을 의미한다.
- 작동 원리
	 1. 여러개의 Corouitne을 이벤트 루프에 한꺼번에 등록
	 2. 첫 번째 작업이 실행되다가 await를 만나 대기 상태(Pending)가 되면, 제어권을 다음 작업으로 넘김
	 3. 이벤트 루프는 각 작업의 대기 시간이 끝나는 순서대로 작업을 다시 깨워 실행을 이어 실행
- 작동 원리2
	- 여러개의 Awaitable 객체를 인자로 받아 하나의 약속 객체로 묶음
	- 모든 작업이 완료 상태가 될 때까지 기다린 후, 결과 값들을 **리스트**형태로 한꺼번에 반환
	- 작업 중 하나라도 실패하면 예외 발생시키거나 설정에 따라 남은 작업을 계속한다

## 헷갈리는 개념 정리
1. yield 함수의 호출자와 async 필수 여부
	1. 보통 다른 함수나 클래스나 프레임워크가 호출자가 된다
	2. 내부에서 await를 쓸 일이 없다면 async를 붙이지 않아도 된다.
2. 약속 객체
	1. await를 작성하지 않으면 약속 객체(Coroutine/Promise)를 받는다
	2. await를 작성하면 실제 결과 데이터를 받는다
3. 약속 객체가 완료 상태로 변하면 await를 작성한 변수에 값을 전달한다
4. Awaitable 객체(부모) $\supset$ Coroutine, Task, Future 객체(자식들)

```python
#약속 객체 받기(await 없음)
#my_obj1은 Coroutine
my_obj1 = httpx.AsyncClient().get("https://....")

#실제 데이터 받기(await 있음)
#rsponse_data는 받은 응답 데이터(예:Http Header,Body 등)
response_data = await httpx.AsyncClient().get("https://...")
```

## 비동기 키워드 정리
| **구분**      | **파이썬 (FastAPI)**     | **자바스크립트 (Node.js/Web)** |
| ----------- | --------------------- | ------------------------ |
| **선언**      | `async def`           | `async function`         |
| **대기**      | `await`               | `await`                  |
| **일시정지/생산** | `yield`               | `yield`                  |
| **약속 객체**   | `Coroutine / Task`    | `Promise`                |
| **동시 실행**   | `asyncio.gather()`    | `Promise.all()`          |
| **네트워크 요청** | `httpx.AsyncClient()` | `fetch()`                |
