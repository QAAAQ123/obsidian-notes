## 주소 설정
```python  
#1. DB 주소 설정(aioslite를 사용해 비동기로 접속)  
DATABASE_URL = "sqlite+aiosqlite:///./sec_analyzer.db"  
```
- String으로 DB주소를 설정한 이유는 표준 형식이 그렇게 정해놓았기 때문이다.
	- 연결의 표준화, 동적 로딩,설정 변경의 유연함 때문에 이렇게 작성한다
		- sqllite를 쓰지 않고 mysql로 변경하고 싶으면 sqlite를 빼고 mysql로 변경만 하면 바뀐다
- `sqlite`: 데이터베이스 종류
- `+aiosqlite`: 비동기 전용 도구로 접속,Thread Pool로 작업 위임
	- 이거 안쓰면 파이썬 기본 모듈인 sqlite3를 사용하는데, 이것은 동기방식이다
- `:///./sec_analyzer.db`: 파일 위치 지정
	- `://`: 구분자
	- `/`: 경로 시작
	- `./sec_analyzer.db`:현재 프로젝트 폴더에 sec_analyzer파일을 만든다


## 엔진 생성
```python
#2. 엔진 생성  
engine = create_async_engine(  
    DATABASE_URL,  
    connect_args={"check_same_thread": False} #SQLite를 비동기로 쓸 때 필수 옵션  
)  
```
- `create_async_engine`: DBMS와 내가 작성한 코드 사이의 연결 관리자를 만듦
	- Connection Pool을 관리하는 기계를 만듦
	- `_async_`를 넣었기 때문에 비동기 엔진임
		- 동기 엔진 생성은 `create_engine`
- `DATABASE_URL`: 데이터 저장할 파일 경로 설정(목적지와 교통수단)
- `connect_args`: 어떤 방법으로 DBMS와 코드를 연결할지 결정
	- `"check_same_thread": False`: SQLite는 쓰레드 하나에서만 처리하는데, 비동기 방식 SQLite를 사용하면 쓰레드를 여러개 쓸 수 밖에 없으니까 동일 쓰레드인지 확인하는 옵션 끔

## 세션 생성기 생성  
```python
#3. 세션 생성기  
#autocommit=False, autoflush=False는 트랜잭션 직접 관리  
AsyncSessionLocal = sessionmaker(  
    bind=engine,  
    class_=AsyncSession,  
    expire_on_commit=False  
)  
```
- `AsyncSessionLocal`: DBMS와 대화할 수 있는 세션을 찍어내는 Factory
	- DBMS연결이 필요할 때 이 AsyncSesisonLocal에서 세션 하나를 받아 쓰는 구조
- `bind=engine`: Factory에서 만드는 세션은 Engine(Connection Pool)을 사용하게 engine과 연결
- `class_=AsyncSession`: 비동기 방식의 문법을 사용한다고 명시
- `expire_on_commit`: 커밋을 해서 저장을 마치면 메모리에 들고 있던 객체를 만료 시킬지 말지 결정
	- False로 하면, 커밋 후에도 에러 없이 메모리에 있는 데이터 조회 가능

## 테이블 metadata 저장소 생성
```python
#4. 모델들의 조상 클래스(Base)

```

- DBMS 테이블을 관리한다

## 서비스 로직에서 db편하게 가져오기 위한 함수 생성
```python
#5. 의존성 주입 함수  
#FastAPI 라우터에서 db를 쓸 때마다 이 함수를 호출해서 세션을 빌려줌  
async def get_db():  
    async with AsyncSessionLocal() as session:  
        try:  
            yield session  
        finally:  
            await session.close() #다 쓰면 연결 끊기
```
- get_db: 요청이 올 때마다 세션을 빌려주고(yield), API가 일을 마칠 때까지 기다렸다가, 일이 끝나면 정리(session.close)해주는 관리인
- DBMS 세션의 `생성-사용-반납`수명주기 관리하기 위해 사용하는 함수/의존성 주입하여 사용한다
- `async with AsyncSessionLocal() as session`: 세션을 하나 가져와서 비동기로 사용할 것이라고 선언
- `yield session`: 비동기로 동작하여 끝날때 까지 기다림
	- yield session에서 멈추면 라우터가 이 세션을 받아서 DB 조회/저장을 수행
- `await session.close()`: 세션 끝나면 무조건 close하게 try-finally를 써서 DB연결을 끊음
