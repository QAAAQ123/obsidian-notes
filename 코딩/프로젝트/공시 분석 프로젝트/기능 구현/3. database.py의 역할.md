## 1. 엔진 생성
- DB서버와 연결된 통신선(Connection Pool)을 관리하는 객체
- Engine의 역할
	1. Connection Pool 관리 주체
		- DB 연결 비용은 매우 큰데, 엔진이 미리 DB와 연결을 해서 메모리에 일정 개수의 연결을 확보해둠
	2. SQL 다언어 변환
		- SQLAlchemy코드를 DBMS가 이해할 수 있는 구체적 SQL문법으로 변환

### Connection Pool
일정 수의 DB 연결을 미리 만들어 메모리에 저장해두는 기술
1. DB 연결/해제 비용 절감
2. 응답 속도 향상
3. 서버 부하 방지

## 2. 세션 생성기(SessionMaker) 생성
- 세션을 만들때 매번 똑같은 옵션들을 적어야 하는 것을 줄이기 위해, 미리 설정값을 저장해둠
- DB와 직접 소통하지 않고, 설계도만 가지고 있음

### 2.1 세션
- Engine에서 관리하는 connection pool중 하나를 빌려서, 실제 작업(CRUD)을 수행하는 임시 공간
- ORM 객체를 추적하고 트랜잭션 단위(commit,rollback)관리
- 가벼움
- 요청이 끝나면 세션 폐기

## 3. DB 테이블 Base 생성
- Base는 테이블 class에 상속되어서 테이블의 기초를 생성한다
- Base의 matadata에 테이블 정보를 저장한다

## 4. 세션 빌려주고 graceful하게 종료하는 함수 생성
- 세션에 대한 구체적인 옵션없이 `세션을 빌려주고 다 쓰면 닫는다`를 쉽게 해주는 함수
- 세션의 생명주기 `생성-사용-반납`을 함수 하나로 처리함
