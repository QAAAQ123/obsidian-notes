## FastAPI의 출시 이유
- Python Framework인 Flask와 Django는 비동기를 완벽하게 지원하지 못했다
- FastAPI는 Uvicorn과 결합하여 완전한 비동기를 구현할 수 있게 되었다
[자세한 내용](https://blog.naver.com/shino1025/223349252829)

## aiosqlite와 httpx
- aiosqlite는 비동기로 sqlite로의 접근을 지원한다
	- sqlite만 사용하면 동기만 사용 가능하다
- httpx는 비동기로 http/https 요청을 할 수 있다.
	- httpx가 아닌 request를 사용하며 동기만 사용 가능하다

## 비동기 라이브러리 사용이유
- 따라서 sec의 html파일이나 AI API를 사용하여 답변을 하는 것 같은 시간이 많이 걸리는 작업을 할 때에
- 사용자가 답답하다고 느끼지 않고 성능을 향상시키기 위해서 FastAPI를 사용했고,FastAPI의 비동기를 최대한 활용하기 위해서 httpx와 aiosqlite 사용한다
