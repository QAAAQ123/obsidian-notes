- 하나의 공시 타입에 여러개의 AI 분석이 들어감(재무따로 위험따로 이런식으로)
- 3개의 테이블: 회사 정보/공시 정보/공시item별 분석 정보
- gemini에게 물어본 결과 위 3개의 테이블로 떨어트려 놓는 것이 **제3정규형을 만족**한다고 함
- 모든 FK는 ID로 할 것

## 회사 정보(Company)
1. id(PK): int
2. ticker: string,unquie
3. name: string
4. cik: string,unquie

## 공시 메타데이터(Filing)
1. id(PK): int
2. accession_number: string,unquie
3. filing_type: string
4. filing_date: Date
5. primary_document: string
- **FK: company_id**

## 분석 내용(Analyses)
1. id: int
2. created_at: DateTime
3. table_of_contents: String
4. content: text(무제한 문자열)
- **FK: filing_id**
## 관계
1. Company:Filing = 1:N
2. Filing:Analyses = 1:N
