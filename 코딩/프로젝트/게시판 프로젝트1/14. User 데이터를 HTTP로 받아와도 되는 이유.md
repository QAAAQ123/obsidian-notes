`/api/join`으로 `UserDTO`를 전송할 때 **HTTP(S)가 자동으로 암호화하여 중간자 공격을 방지**해줍니다. 따라서 컨트롤러에서 `UserDTO`를 받아 처리해도 안전합니다.

---

### HTTPS의 역할

HTTPS는 클라이언트와 서버 간의 통신 채널을 암호화합니다. 이 과정은 다음과 같이 작동합니다.

1. 클라이언트는 서버에 접속하며 서버의 **공개 키**를 받습니다.
    
2. 클라이언트는 이 공개 키로 `UserDTO`의 **평문 데이터**를 암호화하여 서버로 보냅니다.
    
3. 서버는 자신의 **비밀 키**로 암호화된 데이터를 복호화하여 `UserDTO`의 내용을 확인합니다.
    

이 과정 덕분에 네트워크를 통해 전달되는 데이터는 암호화된 상태로 유지되며, 해킹 공격으로부터 보호받습니다.

---

### 서버의 추가 보안 조치

HTTPS 암호화 외에도, 서버에서는 다음과 같은 추가적인 보안 조치를 취해야 합니다.

- **비밀번호 해시화**: `UserDTO`를 받은 후, `BCryptPasswordEncoder`와 같은 해시 함수로 평문 비밀번호를 해시화하여 데이터베이스에 저장합니다.
    
- **비밀번호 재사용 방지**: 사용자가 이전에 사용했던 비밀번호는 재사용하지 못하도록 하는 정책을 적용할 수 있습니다.
    
- **세션 관리**: 인증 성공 후에는 안전한 세션 관리 정책을 적용하여 사용자의 인증 상태를 유지합니다.
