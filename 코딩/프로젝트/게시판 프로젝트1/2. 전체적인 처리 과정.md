1. 클라이언트에서 HTTP GET,POST,PUT,DELETE 요청으로 HTTP BODY에 JSON데이터를 서버 측에 보냄
2. Controller는 @RequestBody와 같은 어노테이션을 통해 ~={red}JSON을 자동으로 DTO로 매핑=~함
3. Serivce는 DTO나 Entity로 비즈니스 로직을 처리함
	1. 비즈니스 로직 처리 과정에서 Repository를 이용해 데이터를 저장 및 조회함
	2. DTO로만 로직을 처리하기도 함
	3. Repository가 메서드를 호출하면 JPA가 영속성 콘텍스트를 활용하여 DB작업을 수행한다
4. Serivce에서 Controller로 처리 완료된 DTO를 보냄
5. Controller는 HTTP 상태 코드와 함께 JSON 형태로 클라이언트에 데이터 보냄  

인증 처리 과정

1. 1의 과정에서 사용자의 아이디,비밀번호가 전송되면 UsernamePasswordAuthenticationFilter(filterChain)이 해당 데이터를 ~={red}가로채서 인증을 시도=~
2. AuthenticationProvider가 ~={red}인증 로직을 통해 비밀번호 확인 등 보안에 필요한 동작=~을 함
	1. 2의 과정에서 UserDetailService객체로 DB에 있는 고객 정보 데이터를 받아옴
3. 처리가 완료된 Authentication 객체는 ~={red}보안 컨텍스트(SecurityContext)에 저장=~된다.
4. ~={red}Controller나 Service 계층에서 접근을 하면=~ 인증 된 사용자 정보를 사용할 수 있도록 한다.(Authentication객체를 Controller나 Service에서 "받아서" 사용하는게 아님)
