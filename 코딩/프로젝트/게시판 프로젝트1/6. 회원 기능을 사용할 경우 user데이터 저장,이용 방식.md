## User 데이터 저장 및 이용 방식
Spring(or Spring Boot)와 JPA를 사용한다고 가정
<br>
### User 데이터 저장 방식
- User 데이터는 ~={red}DB의 테이블이나 JPA의 영속성 콘텍스트(Entity)에 저장=~한다.
- Session이나 Token방식은 민감한 데이터 탈취 방지를 위한 것이기 때문에,서버 쪽의 DB나 영속성 콘텍스트에 저장하는 것은 Session,Token을 사용하는 목적과 부합한다.
<br>
### User 데이터 이용 방식
- Service와 Controller 계층에서 DB와 Persistence Context에 저장된 userId(user데이터)를 이용해도 된다.
- 하지만 ==★★★**user데이터를 DTO,JSON에 넣어 서버-클라이언트간 전송하면 안된다**★★★==. User데이터는 상호 전송시에 ~={red}반드시 별도의 공간(Session,Token)에 담겨서 이동=~해야 한다.
