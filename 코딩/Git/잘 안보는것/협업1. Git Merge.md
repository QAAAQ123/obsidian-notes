## 병합
Git에서 커밋을 결합 할 수 있는 방법은 여러가지가 존재한다.
어떤 Branch를 기준으로 결합하는지에 따라 결과가 다르다
1. fast-forward
2. conflict
3. ...나중에 추가

## ~={red}★★★★★협업시 병합 주의사항★★★★★=~
사전 병합 권장
- 팀원 모두가 공유하는 메인 branch에 병합하기 전에 **~={red}나만 사용하는 브랜치에 먼저 병합을 하는 것을 권장한다=~**
- 병합시 충돌이 발생하면 Conflict가 발생했다는 내용이 Commit에 남게 된다. 
- 메인 브랜치에 충돌 기록이 남으면 모두가 볼 수 있기 때문에, 나만 사용하는 브랜치에 병합을 해보고 메인 branch에 병합을 하는 것이 좋다.

병합 순서
- **~={red}main branch의 최신 내용을 pull=~**->자신의 브랜치에 merge -> 문제가 없으면 main branch에 push


### Fast-forward 병합(빨리감기)
- Fast-forward 병합은 2개의 브랜치를 병합할때 Conlict 하지 않는 경우이다
- 이 경우에는 단순히 HEAD의 위치를 옮기고 병합 된 Branch를 삭제하면 끝난다
- 별도의 병합 커밋이 생성되지 않는다.

### Conflict가 발생한 병합
- Conflict가 발생하면 사용자는 충돌이 발생한 부분의 코드를 수정한 후에 커밋해야 한다.
- Conflict가 발생하면 merge commit 기록에 conflict가 발생했다는 기록이 남는다
