1. 커밋 메시지 잘못 작성했을 때 수정: git commit --amend
   수정한 커밋 메시지를 push하려면 강제로 덮어씌워야 한다. ~={red}강제 덮어씌우기는 반드시 혼자만 쓰고있는 브랜치에서 해야한다=~. 공유 브랜치에 강제 push를 하면 다른 사람의 커밋 히스토리가 꼬인다.
2. 커밋 하나만 떼서 붙히기(해당 커밋의 변경점만 반영): git chery-pick
3. 커밋 되돌리기(HEAD 움직임): git reset
	1. Soft: 커밋 기록(HEAD)만 되돌리고, **작업 디렉토리**와 **스테이징 영역**의 내용은 그대로 유지
	2. Mixed: 커밋 기록(HEAD)과 **스테이징 영역**을 되돌리고, **작업 디렉토리**의 내용은 그대로 유지/기본값/**~={red}언스테이징=~**이라고도 한다.
	3. Hard: 커밋 기록(HEAD), **스테이징 영역**, 그리고 **작업 디렉토리**의 내용을 모두 지정된 커밋 상태로 되돌림
4. 커밋의 변경사항을 되돌리는 새로운 커밋 만들기: git revert
   ex) 싫어요 기능을 없애면서 'Revert: 싫어요 기능 제거'와 같이 명시적으로 삭제한것을 알리고 싶을때 사용,git reset을 통해 싫어요 기능을 없애면 싫어요 기능이 삭제되었는지 모르는 사람이 생길 수 있다.
5. 아직 commit하지 않은 변경 사항을 임시 저장소에 넣어두기: git stash
   ex) hot fix를 해야할때 이전까지 작업한 내역이 commit하기에 애매한 상태일때 작업 내역을 stash에 저장해두고 hot fix를 한 다음 다시 stash에서 꺼내와 원래 하던 작업 진행
