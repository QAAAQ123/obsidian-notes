## git fork
- 남의 Repository를 내 계정의 Repository로 복사해오는 명령어
- branch에서 pull request를 보내는 것과 마찬가지로 원격 저장소에서 원본 저장소로 pull request를 보낼 수 있다.

### fork와 branch 비교
| | 의의 | 편리한 점 | 불편한 점 |
|---|---|---|---|
| 브랜치 | 하나의 원본 저장소에서 분기를 나눈다 | 하나의 원본저장소에서 코드 커밋 이력을 볼 수 있다. | 다수의 사용자가 다수의 브랜치를 만들면 관리하기 어렵다. |
| 포크 | 여러 원격 저장소를 만들어 분기한다 | 원본저장소에 영향을 미치지 않으므로 마음껏 수정할 수 있다. | 원본저장소의 이력을 보려면 따로 주소를 추가해야 한다. |

## 원격 저장소에서 원본 저장소의 커밋 기록 보기
- 원격 저장소로 fork한 시점 이후로 원본 저장소에서 무슨 일이 일어났는지 확인할 수 없다.
- add remote를 하여 원본 저장소의 커밋 기록과 원격 저장소의 커밋 기록을 함께 볼 수 있다.
- **~={red}remote하려는 원본 저장소의 이름은 'upstream'으로 하는것이 관용적이다.=~**
