

### 🔄 Git vs SVN: 핵심 차이점

|항목|Git (분산형)|SVN (중앙집중형)|
|---|---|---|
|저장 방식|**분산 버전 관리**: 모든 사용자가 전체 저장소를 복제|**중앙 집중형**: 서버에만 전체 저장소 존재|
|커밋 저장 방식|커밋 시 **전체 프로젝트의 스냅샷(snapshot)**을 저장.실제로는 변경된 부분만 저장하고, 나머지는 이전 커밋을 참조하는 방식으로 처리|커밋 시 **변경된 파일만 서버에 반영**.이전 버전과의 **차이(diff)** 중심으로 관리|
|오프라인 작업|가능 (로컬에서 커밋, 브랜치 등)|제한적 (서버 연결 필요)|
|속도|빠름 (로컬 작업 중심)|느릴 수 있음 (서버 의존)|
|브랜치 관리|유연하고 빠름|상대적으로 복잡하고 느림|
|병합(Merge)|강력한 병합 기능|병합 기능은 제한적|
|협업 방식|분산된 팀 환경에 적합|중앙 서버 기반 협업에 적합|
